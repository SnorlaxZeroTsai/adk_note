## 從入門到資深的練習路徑

### Level 1：基礎（1-2 天）

#### 練習 1.2：帶工具的 Agent

```python
def get_current_time() -> str:
    """取得目前時間"""
    from datetime import datetime
    return datetime.now().strftime("%Y-%m-%d %H:%M:%S")

def calculate(expression: str) -> str:
    """計算數學表達式"""
    return str(eval(expression))  # 生產環境請用安全的計算方式

root_agent = Agent(
    name="utility_agent",
    model="gemini-2.5-flash",
    instruction="你是工具助理。需要時間就用 get_current_time，需要計算就用 calculate。",
    tools=[get_current_time, calculate],
)
```

**目標：**
- 理解 FunctionTool 的自動包裝
- 觀察工具被呼叫的 Event 流
- 理解 LLM 如何決定要不要用工具

#### 練習 1.3：有狀態的 Agent

```python
def add_to_cart(item: str, tool_context) -> str:
    """加入購物車"""
    cart = tool_context.state.get("cart", [])
    cart.append(item)
    tool_context.state["cart"] = cart
    return f"已加入 {item}，購物車共 {len(cart)} 件"

def view_cart(tool_context) -> str:
    """查看購物車"""
    cart = tool_context.state.get("cart", [])
    if not cart:
        return "購物車是空的"
    return "購物車：" + ", ".join(cart)

root_agent = Agent(
    name="shopping_agent",
    model="gemini-2.5-flash",
    instruction="你是購物助理。幫用戶管理購物車。",
    tools=[add_to_cart, view_cart],
)
```

**目標：**
- 理解 ToolContext 的使用
- 觀察 State 的變化
- 理解 state_delta 在 Event 中的表現

### Level 3：生產級系統（1-2 週）

#### 練習 3.1：回呼系統

```python
import time

call_log = []

def log_model_call(callback_context, llm_request):
    """記錄每次 LLM 呼叫"""
    call_log.append({
        "time": time.time(),
        "model": llm_request.model,
        "tokens_estimate": len(str(llm_request.contents)),
    })
    return None  # 繼續正常流程

def cost_guard(callback_context, llm_request):
    """成本控制：超過 10 次呼叫就停止"""
    if len(call_log) >= 10:
        from google.adk.models import LlmResponse
        return LlmResponse(content="已達到呼叫上限，請稍後再試。")
    return None

root_agent = Agent(
    name="guarded_agent",
    before_model_callback=cost_guard,
    after_model_callback=log_model_call,
    ...
)
```

**目標：**
- 實作成本控制
- 實作日誌記錄
- 理解回呼的攔截機制

#### 練習 4.1：自訂 SessionService

```python
from google.adk.sessions import BaseSessionService, Session

class RedisSessionService(BaseSessionService):
    """用 Redis 做 Session 儲存"""
    
    def __init__(self, redis_url: str):
        self.redis = Redis.from_url(redis_url)
    
    async def create_session(self, app_name, user_id, **kwargs) -> Session:
        session = Session(id=uuid4(), app_name=app_name, user_id=user_id)
        await self.redis.set(f"session:{session.id}", session.json())
        return session
    
    async def get_session(self, app_name, user_id, session_id) -> Session:
        data = await self.redis.get(f"session:{session_id}")
        return Session.parse_raw(data)
    
    # ... 其他方法
```

**目標：**
- 理解 Service 抽象的設計意圖
- 實作自己的 Service
- 處理序列化/反序列化
- 考慮並發和一致性

#### 練習 4.2：自訂 Tool with Auth

```python
from google.adk.tools import BaseTool
from google.adk.auth import AuthConfig

class SlackTool(BaseTool):
    """發送 Slack 訊息的工具"""
    
    name = "send_slack_message"
    description = "發送訊息到 Slack 頻道"
    
    def get_auth_config(self):
        return AuthConfig(
            auth_type="oauth2",
            client_id="your-client-id",
            scopes=["chat:write"],
        )
    
    async def run_async(self, args, tool_context):
        # 取得認證後的 token
        creds = tool_context.get_auth_response()
        
        # 呼叫 Slack API
        async with httpx.AsyncClient() as client:
            response = await client.post(
                "https://slack.com/api/chat.postMessage",
                headers={"Authorization": f"Bearer {creds.token}"},
                json={"channel": args["channel"], "text": args["message"]},
            )
        return response.json()
```

**目標：**
- 實作完整的自訂工具
- 處理 OAuth 認證流程
- 理解認證在 ADK 中的生命週期

## 資深 Agent 工程師的思維清單

每次設計 Agent 系統時，問自己：

- [ ] Agent 的職責是否單一且明確？
- [ ] 工具的描述是否讓 LLM 能正確判斷何時使用？
- [ ] State 的生命週期是否合理？（temp/session/user/app）
- [ ] 有沒有做成本控制？（回呼限制呼叫次數）
- [ ] 有沒有評估測試集？
- [ ] 串流是否正確實作？（用戶體驗）
- [ ] 錯誤處理是否完善？（LLM 故障、工具故障）
- [ ] 部署策略是否匹配需求？
- [ ] 記憶策略是否合理？（該記什麼，不該記什麼）
- [ ] 安全性？（工具的權限控制、輸入驗證）

## 結語

成為資深 Agent 工程師不只是會用框架，而是：
1. **理解設計決策背後的原因** — 為什麼這樣設計而不是那樣
2. **能做架構權衡** — 知道每個選擇的代價
3. **能設計可維護的系統** — 可測試、可監控、可擴展
4. **能處理真實世界的複雜性** — 認證、錯誤、成本、安全

ADK 的程式碼本身就是最好的學習材料。讀原始碼，理解每個抽象層的設計意圖，這比任何教程都有用。
