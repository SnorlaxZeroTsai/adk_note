# 01 - 核心設計哲學

## ADK 是什麼？

Google Agent Development Kit (ADK) 是一個 **以程式碼為核心** 的 Python 框架，用來建構、評估、部署 AI Agent。

簡單說：它是 Google 版的「Agent 建構框架」，讓你用 Python 程式碼定義 Agent 的行為，而不是用 GUI 拖拉。

## 三大設計原則

### 1. Code-First（程式碼優先）

```python
# 所有東西都是 Python 程式碼
root_agent = Agent(
    name="my_agent",
    model="gemini-2.5-flash",
    instruction="你是一個有幫助的助理",
    tools=[search_tool, calculator_tool],
)
```

**為什麼這樣設計？**
- 可以用 Git 做版本控制
- 可以寫單元測試
- IDE 支援（自動補全、型別檢查）
- 可以 Code Review

**資深工程師的觀點：** 這和 Infrastructure as Code 的哲學一樣。當你的 Agent 邏輯複雜到一定程度，GUI 會成為瓶頸。程式碼讓你保留完整的審計軌跡。

### 2. Modularity & Composition（模組化與組合）

ADK 不鼓勵你寫一個「超級大 Agent」，而是把複雜系統拆成多個小型專業 Agent，再組合起來。

```
Root Agent (路由器)
├── Search Agent (負責搜尋)
├── Analysis Agent (負責分析)
└── Writer Agent (負責撰寫)
```

**為什麼這樣設計？**
- 單一職責：每個 Agent 只做一件事，做好一件事
- 可測試：可以單獨測試每個 Agent
- 可替換：換掉一個子 Agent 不影響其他
- 可重複使用：同一個 Agent 可以在不同系統中使用

**資深工程師的觀點：** 這就是微服務架構的思維應用在 Agent 上。小 Agent = 微服務，組合 = 服務編排。

### 3. Deployment-Agnostic（部署無關）

同一份 `agent.py` 可以：
- 在本地跑測試 (`adk run`)
- 透過 API 提供服務 (`adk web`)
- 部署到雲端 (`adk deploy`)

**為什麼這樣設計？**
- 開發環境和生產環境用同一份程式碼
- 不需要為了部署重寫邏輯
- 降低「在我機器上可以跑」的問題

## 核心抽象概念（你的詞彙表）

| 概念 | 類比 | 說明 |
|------|------|------|
| **Agent** | 員工 | 定義身份、指令、能力的藍圖 |
| **Tool** | 工具箱 | Agent 可以呼叫的能力（函式） |
| **Runner** | 主管 | 協調 Agent 的推理-行動循環 |
| **Session** | 對話紀錄 | 一次連續對話的狀態 |
| **Memory** | 長期記憶 | 跨不同 Session 的回憶 |
| **Artifact** | 附件 | 非文字資料（檔案、圖片） |
| **Workflow** | 流程圖 | 確定性的圖結構執行引擎 |
| **Event** | 紀錄條目 | 系統中發生的每一件事 |

## 專案標準結構

```
my_agent/
├── __init__.py   # 必須包含: from . import agent
└── agent.py      # 必須定義: root_agent = Agent(...)
```

**關鍵規則：**
- 進入點變數必須命名為 `root_agent`
- `__init__.py` 必須匯入 agent 模組
- 這是 ADK 工具鏈（CLI、Web UI）找到你 Agent 的方式

## 為什麼要學這個框架？

作為資深 Agent 工程師，你需要理解：

1. **Agent 不只是一個 Prompt** — 它是一個有狀態、有工具、有記憶的系統
2. **生產環境需要工程紀律** — 測試、版本控制、監控、部署管線
3. **複雜系統需要好的抽象** — ADK 的抽象層設計是學習的典範
4. **多 Agent 協作是未來** — 單一 Agent 有極限，組合才是答案
