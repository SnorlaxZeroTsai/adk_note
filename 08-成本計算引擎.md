# 第八章：成本計算引擎

> **本章目標**：理解 LiteLLM 如何計算每次 API 呼叫的成本，以及如何用這個知識管理你的 AI 應用預算。
>
> **預計閱讀時間**：20 分鐘
>
> **為什麼這章重要**：AI API 是「用多少付多少」的。不像 SaaS 軟體是固定月費——你的程式碼有個 bug 導致無限迴圈呼叫 API，一晚上就可能燒掉幾千美元。理解成本計算是保護你錢包的第一步。
>
> **關聯章節**：[第 14 章：Token Budget 策略](./14-Token-Budget-策略與MCP實作.md) 會教你如何用這些知識做預算規劃。

## 為什麼成本計算重要？

AI 應用的成本管理是企業級部署的關鍵。不同模型、不同供應商、甚至同一模型的不同操作（輸入 vs 輸出、快取 vs 非快取）都有不同的定價。LiteLLM 提供了精確到每次呼叫的成本追蹤。

> **真實案例**：一家新創在使用 GPT-4 做客服機器人時，因為沒有設定 `max_tokens`，某些使用者的提問觸發了超長回覆，導致單一使用者的一次對話就花了 $2。乘以 10,000 活躍使用者，月帳單從預期的 $500 飆到 $15,000。成本計算 + 預算告警本可以在第一天就攔住這個問題。

## completion_cost() 函式

這是成本計算的主入口：

```python
# litellm/cost_calculator.py

def completion_cost(
    completion_response=None,    # ModelResponse 物件
    model: Optional[str] = None,
    prompt: str = "",
    messages: List = [],
    completion: str = "",
    total_time: Optional[float] = 0.0,    # 用於按時間計費的模型
    call_type: Optional[str] = None,       # "completion", "embedding", "image_generation" 等
    custom_llm_provider=None,
    region_name=None,            # 區域定價差異
    size=None,                   # 圖片尺寸（圖片生成）
    quality=None,                # 圖片品質
    custom_cost_per_token=None,  # 自定義定價
) -> float:
    """
    計算一次 LLM 呼叫的美元成本。
    
    Returns:
        float: 成本（美元）
    """
```

### 計算流程

```python
def completion_cost(completion_response, model, ...):
    # 1. 從回應中提取 token 用量
    if completion_response:
        usage = completion_response.usage
        prompt_tokens = usage.prompt_tokens
        completion_tokens = usage.completion_tokens
        model = completion_response.model
    else:
        # 沒有回應時，用 tokenizer 估算
        prompt_tokens = token_counter(model=model, text=prompt)
        completion_tokens = token_counter(model=model, text=completion)
    
    # 2. 取得模型定價
    model_info = get_model_info(model=model, custom_llm_provider=provider)
    
    # 3. 根據供應商計算成本
    if custom_llm_provider == "anthropic":
        cost = anthropic_cost_per_token(
            model=model,
            prompt_tokens=prompt_tokens,
            completion_tokens=completion_tokens,
            cache_creation_tokens=cache_creation_tokens,
            cache_read_tokens=cache_read_tokens,
        )
    elif custom_llm_provider == "openai":
        cost = openai_cost_per_token(
            model=model,
            prompt_tokens=prompt_tokens,
            completion_tokens=completion_tokens,
        )
    else:
        # 通用計算
        cost = generic_cost_per_token(
            model=model,
            prompt_tokens=prompt_tokens,
            completion_tokens=completion_tokens,
            model_info=model_info,
        )
    
    return cost
```

## 通用 Token 計費公式

```python
def generic_cost_per_token(model, prompt_tokens, completion_tokens, model_info):
    """
    成本 = (input_tokens × input_price) + (output_tokens × output_price)
    """
    input_cost = prompt_tokens * model_info["input_cost_per_token"]
    output_cost = completion_tokens * model_info["output_cost_per_token"]
    
    return input_cost + output_cost
```

## 供應商特定的成本計算

### Anthropic — 快取定價

Anthropic 的 prompt caching 有特殊定價：

```python
def anthropic_cost_per_token(model, prompt_tokens, completion_tokens, 
                             cache_creation_tokens, cache_read_tokens):
    """
    Anthropic 有三種 input 定價:
    1. 普通 input tokens     → input_cost_per_token
    2. 快取建立 tokens       → cache_creation_input_token_cost（1.25x）
    3. 快取讀取 tokens       → cache_read_input_token_cost（0.1x）
    """
    model_info = get_model_info(model)
    
    # 普通 input（扣除快取的部分）
    regular_input = prompt_tokens - cache_creation_tokens - cache_read_tokens
    input_cost = regular_input * model_info["input_cost_per_token"]
    
    # 快取建立（更貴，因為要存儲）
    cache_create_cost = cache_creation_tokens * model_info["cache_creation_input_token_cost"]
    
    # 快取讀取（便宜很多）
    cache_read_cost = cache_read_tokens * model_info["cache_read_input_token_cost"]
    
    # Output
    output_cost = completion_tokens * model_info["output_cost_per_token"]
    
    return input_cost + cache_create_cost + cache_read_cost + output_cost
```

### OpenAI — 服務層級定價

```python
def openai_cost_per_token(model, prompt_tokens, completion_tokens, service_tier=None):
    """
    OpenAI 的 'flex' 服務層級有折扣定價
    """
    model_info = get_model_info(model)
    
    if service_tier == "flex":
        # flex tier 有特定的定價 key
        input_price = model_info.get("flex_input_cost_per_token", 
                                     model_info["input_cost_per_token"])
        output_price = model_info.get("flex_output_cost_per_token",
                                      model_info["output_cost_per_token"])
    else:
        input_price = model_info["input_cost_per_token"]
        output_price = model_info["output_cost_per_token"]
    
    return (prompt_tokens * input_price) + (completion_tokens * output_price)
```

### Bedrock — 區域定價

```python
def bedrock_cost_per_token(model, prompt_tokens, completion_tokens, region_name):
    """
    AWS Bedrock 的定價因區域而異
    """
    model_info = get_model_info(model)
    
    # 區域加成（某些區域比 us-east-1 貴）
    regional_multiplier = _get_regional_uplift_multiplier(
        model=model,
        region=region_name,
    )
    
    base_cost = generic_cost_per_token(model, prompt_tokens, completion_tokens, model_info)
    return base_cost * regional_multiplier
```

## 圖片生成的成本

圖片模型不按 token 計費，而是按圖片數量、尺寸、品質：

```python
def _image_generation_cost(model, size, quality, n):
    """
    DALL-E 3 定價範例：
    - 1024x1024, standard: $0.040
    - 1024x1024, hd: $0.080
    - 1792x1024, standard: $0.080
    - 1792x1024, hd: $0.120
    """
    price_per_image = get_image_price(model, size, quality)
    return price_per_image * n
```

## Token 計數

LiteLLM 使用 `tiktoken`（OpenAI 的 tokenizer）來計算 token 數量：

```python
from litellm import token_counter

# 計算 messages 的 token 數
count = token_counter(
    model="gpt-4",
    messages=[{"role": "user", "content": "Hello world"}]
)
# → 約 12 tokens（包含格式化 overhead）
```

不同模型使用不同的 tokenizer：
- GPT-4/3.5：`cl100k_base`
- GPT-4o：`o200k_base`
- Claude：使用 Anthropic 的計數 API
- 其他：回退到 `cl100k_base` 作為估算

## 學習重點

1. **定價資料庫** (`model_prices_and_context_window.json`) 是所有成本計算的基礎
2. **通用公式**：`cost = input_tokens × input_price + output_tokens × output_price`
3. **供應商特殊邏輯**：快取定價、區域加成、服務層級折扣
4. **成本流動**：從 completion → logging → callback → 使用者/資料庫
5. **自定義定價**允許使用者覆蓋任何模型的預設價格

## 快速心算技巧

記住這些近似值，讓你能快速心算成本：

```
GPT-4o-mini：≈ $0.001 / 每千次簡單呼叫（1K in + 500 out）
GPT-4o：     ≈ $0.015 / 每次呼叫（2K in + 1K out）
Claude Opus：≈ $0.105 / 每次呼叫（2K in + 1K out）

月度估算公式（口算版）：
月成本 ≈ 每次成本 × 日呼叫次數 × 30
```

**範例**：你的 app 每天有 1000 次呼叫，用 GPT-4o：
- 月成本 ≈ $0.015 × 1000 × 30 = **$450/月**
- 如果改用 GPT-4o-mini：$0.001 × 1000 × 30 = **$30/月**（省 93%！）

## 自我檢測

1. 一次使用 Claude Opus 的呼叫，2000 input tokens + 1000 output tokens，成本是多少？
2. 如果啟用 prompt caching（90% cache hit），相同的呼叫成本降為多少？

<details>
<summary>答案</summary>

1. (2000 × $15/1M) + (1000 × $75/1M) = $0.03 + $0.075 = **$0.105**
2. 假設 1800 tokens 命中快取（$0.30/1M）、200 未命中（$15/1M）：
   - Input: (1800 × $0.30/1M) + (200 × $15/1M) = $0.00054 + $0.003 = $0.00354
   - Output: 1000 × $75/1M = $0.075
