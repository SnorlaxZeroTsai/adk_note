## 1. Corpus2Skill: Don't Retrieve, Navigate (arXiv: 2604.14572)

### 論文標題
Corpus2Skill: Don't Retrieve, Navigate

### 中文摘要
本論文提出一種全新的知識檢索範式，將傳統的語料庫檢索（retrieval）替換為技能樹遍歷（skill-tree traversal）。系統將整個語料庫編譯為層次化的 Agent Skills 樹結構，使代理能夠透過導航而非搜索來存取知識。這種方法避免了傳統 RAG 系統中向量相似度搜索的局限性，提供了更具結構性和可解釋性的知識存取方式。

### 核心貢獻
- 提出「導航取代檢索」的新範式，從根本上改變了知識存取的方式
- 設計了將非結構化語料庫自動編譯為層次化技能樹的方法
- 技能樹結構提供了更好的可解釋性，使用者能清楚了解知識的組織方式
- 減少了對嵌入模型品質的依賴，避免語義搜索中的「語義鴻溝」問題
- 支援增量更新，新知識可以作為新的技能節點插入樹中

### 方法論
1. **語料庫分析與分解**：將輸入語料庫分解為獨立的知識單元
2. **層次化組織**：根據知識單元之間的語義關係和依賴關係，建構多層次的技能樹
3. **技能節點編譯**：每個葉節點封裝一個可執行的「技能」（skill），包含知識摘要和操作指令
4. **樹遍歷機制**：Agent 從根節點出發，根據查詢語義逐層向下導航，選擇最相關的子節點
5. **動態路徑選擇**：利用 LLM 的推理能力在每個分支點做出導航決策

### 主要發現
- 層次化導航在多跳問答任務中優於傳統的平面檢索方法
- 技能樹結構在複雜推理任務中表現尤為突出，因為它保留了知識之間的邏輯關係
- 導航過程的計算開銷與樹的深度（對數級）相關，而非語料庫大小（線性級）
- 該方法特別適合需要多步推理的場景，因為樹結構天然支援知識的逐步展開

**研究限制:**
- 技能樹的初始編譯過程依賴大量 LLM 呼叫，對於超大規模語料庫的建構成本和時間可能極高，可擴展性存疑
- 評估主要集中於多跳問答任務，缺乏在其他知識密集型任務（如摘要、事實查核）上的驗證
- 樹結構的品質高度依賴自動化分解和組織步驟的準確性，錯誤會在層次中傳播且難以修正

**其他資訊:**
- 與 GraphRAG 等基於圖結構的檢索方法互補，兩者皆嘗試以結構化方式取代平面向量檢索
- 開放問題：當語料庫頻繁更新時，增量更新的效率和樹結構退化問題尚未充分探討

## 2. BudgetMem: Query-Aware Budget-Tier Routing for Agent Memory (arXiv: 2602.06025)

### 論文標題
BudgetMem: Query-Aware Budget-Tier Routing for Agent Memory

### 中文摘要
BudgetMem 提出了一種查詢感知的預算分層路由機制，用於代理記憶系統。系統根據查詢的難度和複雜度，將記憶查詢路由到不同的處理層級（tier），從簡單的鍵值查找到複雜的推理增強檢索。這種方法在保持回答品質的同時，顯著降低了計算成本和延遲。

### 核心貢獻
- 提出查詢難度感知的分層路由架構，實現計算資源的最優分配
- 設計了多個處理層級，從低成本的精確匹配到高成本的深度推理
- 引入了難度評估模組，能夠快速判斷查詢所需的處理複雜度
- 證明大部分記憶查詢可以用低成本方法解決，僅少數需要複雜處理
- 提供了在效能與成本之間的可調節平衡機制

### 方法論
1. **查詢分類器（Query Classifier）**：輕量級模型快速評估查詢難度
2. **多層級處理管線**：
   - **Tier 1（低成本）**：直接鍵值匹配或簡單向量檢索
   - **Tier 2（中等成本）**：帶重排序的檢索增強
   - **Tier 3（高成本）**：多步推理或 LLM 增強的記憶合成
3. **路由決策**：基於查詢特徵（長度、複雜度、歧義性）做出路由決定
4. **回饋學習**：根據下游任務表現持續優化路由策略

### 主要發現
- 約 60-70% 的記憶查詢可以在最低層級得到滿意解答
- 分層路由將整體計算成本降低 40-60%，同時品質損失小於 5%
- 查詢難度與最優處理層級之間存在強相關性
- 動態路由優於靜態的「一刀切」方法，尤其在查詢分佈不均勻時

**研究限制:**
- 查詢分類器本身的訓練需要大量標註數據，且分類錯誤會導致品質損失或資源浪費
- 評估基準可能未涵蓋真實生產環境中的查詢分佈多樣性，泛化能力有待驗證
- 分層邊界的設定較為靜態，對於不同應用場景可能需要重新校準

**其他資訊:**
- 與級聯推理（cascade inference）和混合專家模型（MoE）的路由思想有相似之處，可借鑑相關領域的路由優化研究
- 應用前景：可整合至大規模對話系統和長期記憶代理中，實現成本可控的記憶管理

## 3. Learning to Share: Selective Memory for Parallel Agentic Systems (arXiv: 2602.05965)

### 論文標題
Learning to Share: Selective Memory for Parallel Agentic Systems

### 中文摘要
本論文針對平行多代理系統中的記憶共享問題，提出了一種帶有學習型控制器的共享記憶庫架構。控制器學習判斷哪些記憶應該被共享、何時共享、以及與誰共享，從而避免資訊過載同時促進有效協作。該系統解決了平行代理團隊中記憶隔離與資訊冗餘之間的核心矛盾。

### 核心貢獻
- 提出選擇性記憶共享框架，解決平行代理系統中的資訊協調問題
- 設計了可學習的共享控制器（sharing controller），自動決定記憶的共享策略
- 建立了記憶相關性評估機制，避免無關記憶的傳播造成干擾
- 證明選擇性共享優於「全部共享」和「完全隔離」兩種極端策略
- 提出了記憶衝突檢測與解決機制

### 方法論
1. **共享記憶庫（Shared Memory Bank）**：集中式存儲，所有代理可讀寫
2. **學習型控制器**：
   - 輸入：記憶內容、產生該記憶的任務上下文、其他代理的當前狀態
   - 輸出：共享決策（是否共享、共享對象、共享時機）
3. **相關性評分**：評估一條記憶對其他代理當前任務的潛在價值
4. **訓練信號**：基於下游任務表現的獎勵信號來訓練控制器
5. **衝突解決**：當多個代理產生矛盾記憶時的仲裁機制

### 主要發現
- 選擇性共享在團隊任務完成率上比全共享提升 15-25%
- 無選擇的全面共享會導致「記憶污染」，降低個體代理的決策品質
- 控制器學會了基於任務相關性和時效性來做共享決策
- 在任務異質性高的場景中，選擇性共享的優勢更加明顯

**研究限制:**
- 實驗中的代理數量有限，當平行代理數量大幅增加時，共享控制器的可擴展性和決策延遲是否仍可接受尚不明確
- 訓練共享控制器的獎勵信號來自特定任務集，跨任務類型的泛化能力未經充分測試
- 集中式共享記憶庫存在單點故障風險，分散式架構的可行性未被探討

**其他資訊:**
- 與聯邦學習中的選擇性模型聚合、分散式系統中的 gossip 協議等有概念上的相似性
- 開放問題：當代理的目標存在競爭關係（而非純合作）時，選擇性共享策略如何調整

## 4. CompactRAG: Reducing LLM Calls in Multi-Hop QA (arXiv: 2602.05728)

### 論文標題
CompactRAG: Reducing LLM Calls in Multi-Hop QA

### 中文摘要
CompactRAG 提出了一種降低多跳問答中 LLM 呼叫次數的高效方法。核心思想是將語料庫預處理為原子化的問答對（atomic QA pairs），使得大多數多跳問題可以僅通過兩次 LLM 呼叫即可解決：一次用於分解問題，一次用於合成最終答案。這大幅降低了傳統迭代檢索方法的計算開銷。

### 核心貢獻
- 提出語料庫預處理為原子 QA 對的策略，將檢索從文檔級提升為答案級
- 將多跳問答的 LLM 呼叫次數從 O(n) 降低到常數級（兩次）
- 設計了自動化的原子 QA 對生成管線
- 在保持回答品質的前提下，顯著降低推理延遲和成本
- 提出了 QA 對品質驗證和去重機制

### 方法論
1. **離線預處理階段**：
   - 將語料庫文檔分解為原子化的事實陳述
   - 為每個原子事實生成對應的問答對
   - 建立 QA 對索引，支援高效檢索
2. **線上推理階段**：
   - **第一次 LLM 呼叫**：將複雜多跳問題分解為原子子問題
   - **檢索**：為每個子問題匹配預生成的 QA 對
   - **第二次 LLM 呼叫**：基於收集到的原子答案合成最終回答
3. **品質保證**：QA 對生成時的一致性檢查和覆蓋率評估

### 主要發現
- 兩次 LLM 呼叫即可解決大部分多跳問題，準確率接近迭代方法
- 推理速度提升 3-5 倍，成本降低 60-80%
- 原子 QA 對的品質是系統性能的關鍵瓶頸
- 對於需要超過 3 跳的極複雜問題，性能有所下降
- 預處理的一次性成本可通過多次查詢攤銷

**研究限制:**
- 離線預處理需要對整個語料庫進行 LLM 處理，初始成本高昂，且語料庫更新時需重新生成 QA 對
- 原子 QA 對的品質直接制約系統上限，自動生成過程中的錯誤難以系統性地檢測和修正
- 對於需要超過 3 跳推理的問題性能下降，說明方法在極複雜推理場景的泛化能力有限

**其他資訊:**
- 與 RAPTOR（遞歸摘要樹）等預處理型 RAG 方法形成互補，兩者可結合使用
- 應用場景：特別適合語料庫相對穩定、查詢量大的客服和知識庫問答系統

## 5. Mitigating Hallucination in Financial RAG via Knowledge Verification (arXiv: 2602.05723)

### 論文標題
Mitigating Hallucination in Financial RAG via Knowledge Verification

### 中文摘要
本論文針對金融領域 RAG 系統中的幻覺問題，提出了基於知識驗證的緩解方法。核心方法是將生成的答案分解為原子事實（atomic facts），然後使用強化學習獎勵機制逐一驗證每個事實的正確性。該方法特別適用於金融場景，因為金融領域對事實準確性的要求極高，任何幻覺都可能導致嚴重後果。

### 核心貢獻
- 提出原子事實分解 + RL 驗證的幻覺緩解框架
- 設計了適用於金融文本的原子事實提取方法
- 引入基於強化學習的事實驗證獎勵模型
- 在金融問答基準上顯著降低了幻覺率
- 提出了金融特定的幻覺分類體系（數值幻覺、時間幻覺、實體幻覺等）

### 方法論
1. **答案生成**：RAG 系統基於檢索到的金融文檔生成初始答案
2. **原子事實分解**：將答案拆分為獨立可驗證的原子事實單元
3. **事實驗證模組**：
   - 將每個原子事實與源文檔進行對比
   - 使用 RL 訓練的驗證模型評估事實的支持程度
   - 輸出每個事實的可信度分數
4. **RL 獎勵設計**：
   - 正獎勵：正確識別有支持的事實
   - 負獎勵：遺漏幻覺事實或誤判正確事實
5. **答案修正**：移除或修正低可信度的原子事實，重新合成答案

### 主要發現
- 幻覺率從基線的 23% 降低到 8% 以下
- 數值相關的幻覺是金融 RAG 中最常見的類型
- RL 獎勵機制比監督學習方法更能泛化到新的金融文檔
- 原子事實級別的驗證比句子級別更精確
- 驗證步驟增加約 30% 的推理時間，但在金融場景中這是可接受的代價

**研究限制:**
- 評估僅限於金融領域，方法在其他高精度要求領域（如醫療、法律）的泛化性未經驗證
- RL 驗證模型的訓練需要高品質的事實標註數據，在新的金融子領域中標註成本高
- 增加 30% 推理時間在即時交易決策等低延遲場景中可能不可接受

**其他資訊:**
- 與 FActScore、SAFE 等事實性評估方法有方法論上的聯繫，可視為將評估方法轉化為線上修正機制
- 開放問題：如何處理金融領域中「合理推斷」與「幻覺」之間的灰色地帶（如基於歷史趨勢的預測性陳述）

## 6. Graph-based Agent Memory: Taxonomy, Techniques, Applications (arXiv: 2602.05665)

### 論文標題
Graph-based Agent Memory: Taxonomy, Techniques, Applications

### 中文摘要
本論文是一篇關於圖結構代理記憶架構的綜述論文。系統性地梳理了利用圖結構（知識圖譜、語義網路、記憶圖等）來組織和管理 AI 代理記憶的各種方法。論文建立了完整的分類體系，涵蓋記憶的存儲、索引、檢索、更新和遺忘等關鍵操作，並分析了不同圖結構在各類應用場景中的優劣。

### 核心貢獻
- 建立了圖結構代理記憶的完整分類體系（taxonomy）
- 系統化比較了不同圖結構（知識圖譜、屬性圖、超圖等）的適用場景
- 總結了圖記憶的核心操作：編碼、存儲、檢索、更新、遺忘
- 分析了圖記憶在對話系統、任務規劃、多代理協作等場景的應用
- 指出了當前研究的挑戰和未來方向

### 方法論（綜述框架）
1. **記憶類型分類**：
   - 情節記憶圖（Episodic Memory Graph）：記錄事件序列和因果關係
   - 語義記憶圖（Semantic Memory Graph）：存儲概念和事實知識
   - 程序記憶圖（Procedural Memory Graph）：編碼操作流程和技能
2. **圖結構分類**：
   - 知識圖譜（三元組形式）
   - 屬性圖（節點和邊帶屬性）
   - 時序圖（帶時間戳的動態圖）
   - 超圖（多元關係）
3. **關鍵操作**：
   - 記憶編碼：如何將經驗轉化為圖結構
   - 記憶檢索：子圖匹配、路徑搜索、圖神經網路
   - 記憶更新：增量式圖更新、衝突解決
   - 記憶遺忘：基於重要性和時效性的修剪

### 主要發現
- 圖結構在表達複雜關係和支援多跳推理方面優於平面記憶
- 時序圖在需要追蹤記憶演變的場景中表現最佳
- 圖記憶的主要瓶頸在於自動化圖構建的品質
- 結合向量檢索和圖遍歷的混合方法是當前最有效的檢索策略
- 記憶圖的可擴展性仍是開放挑戰，大規模圖的維護成本高

**研究限制:**
- 作為綜述論文，缺乏統一的實驗基準對各方法進行公平比較
- 對圖記憶的計算和存儲開銷缺乏系統性的量化分析，難以指導實際系統選型
- 所涵蓋的方法多為近期研究，長期穩定性和生產環境中的可靠性尚未得到驗證

**其他資訊:**
- 該綜述為圖記憶研究提供了良好的入門地圖，建議結合 MemoryBank、MemGPT 等具體系統論文深入閱讀
- 開放問題：圖記憶與神經符號推理（neuro-symbolic reasoning）的結合是否能解決自動化圖構建的品質問題

## 7. AI Agent Systems for Supply Chains (arXiv: 2602.05524)

### 論文標題
AI Agent Systems for Supply Chains

### 中文摘要
本論文探討了 AI 代理系統在供應鏈管理中的應用，特別是利用過去類似決策的檢索來輔助庫存管理。系統透過記憶過去的決策情境和結果，在面對新的庫存決策時檢索最相似的歷史案例，結合當前情境進行推理，從而做出更好的庫存決策。

### 核心貢獻
- 將 RAG 範式應用於供應鏈庫存管理的決策支援
- 設計了決策案例的結構化表示方法，便於相似性檢索
- 提出了情境感知的決策檢索機制，考慮供應鏈特有的時間性和季節性
- 證明了歷史決策檢索能顯著提升庫存管理的效率
- 建立了供應鏈決策的評估框架

### 方法論
1. **決策記憶庫構建**：

- 長期歷史數據的積累持續改善決策品質
- 可解釋性是供應鏈場景的重要優勢——決策可追溯到歷史案例

**研究限制:**

**其他資訊:**
- 與案例推理（Case-Based Reasoning, CBR）傳統方法有明確的學術淵源，結合 LLM 推理能力是重要的現代化升級
- 應用前景：可擴展至其他需要歷史經驗指導的營運決策場景，如定價策略和物流路線規劃

## 10. Aggregation Queries over Unstructured Text (arXiv: 2602.01355)

### 論文標題
Aggregation Queries over Unstructured Text

### 中文摘要
本論文提出了一種代理式方法來處理非結構化文本上的聚合查詢（如「有多少文檔提到了...」「最常見的原因是什麼」）。方法分為三個階段：消歧（disambiguation）、過濾（filtering）和聚合（aggregation），使 LLM 能夠有效處理傳統上需要結構化數據庫才能回答的聚合型問題。

### 核心貢獻
- 提出針對非結構化文本聚合查詢的系統化三階段方法
- 設計了查詢消歧機制，處理聚合查詢中的模糊性
- 提出了高效的文檔過濾策略，減少不必要的 LLM 處理
- 支持計數、排名、分組、趨勢分析等多種聚合操作
- 在準確性和效率之間取得了良好平衡

### 方法論
1. **消歧階段（Disambiguation）**：
   - 分析聚合查詢的意圖和範圍
   - 明確聚合的對象、條件和粒度
   - 處理語義模糊性（如「近期」「相關」的定義）
2. **過濾階段（Filtering）**：
   - 快速篩選與查詢相關的文檔子集
   - 使用輕量級分類器減少需要深度處理的文檔數量
   - 支持多輪漸進式過濾
3. **聚合階段（Aggregation）**：
   - 從過濾後的文檔中提取結構化資訊
   - 執行聚合操作（計數、分組、排序等）
   - 生成自然語言的聚合結果摘要

### 主要發現
- 三階段方法在聚合準確率上比端到端方法提升 25-35%
- 消歧步驟對最終品質影響最大——模糊查詢是主要錯誤來源
- 過濾階段可以減少 70-80% 的文檔處理量
- 對於需要精確計數的查詢，準確率仍有改善空間
- 方法在法律、醫療等文檔密集型領域特別有價值

**研究限制:**
- 精確計數類查詢的準確率仍不理想，LLM 在大規模文檔集上的計數一致性難以保證
- 對超大規模文檔集（百萬級以上）的處理效率和成本未經充分評估
- 消歧階段的效果依賴 LLM 的語義理解能力，對於高度專業化的領域術語可能需要額外適配

**其他資訊:**
- 與 Text2SQL、LOTUS 等結構化查詢生成方法互補，前者處理已結構化數據，本方法處理純文本
- 應用前景：法律發現（legal discovery）、市場調研報告自動生成、大規模文獻計量分析

- 反思機制能有效識別答案的覆蓋盲區

- 對於具有多個合理答案面向的問題，改善最為顯著

- 迭代精煉增加了延遲和 LLM 呼叫成本，在即時回應場景中可能不適用
- 多樣性的評估指標本身具有主觀性，不同用戶對「全面覆蓋」的期望可能不同
- 對於有明確單一正確答案的事實性問題，多樣性增強可能引入不必要的噪音

**其他資訊:**
- 與搜索引擎中的結果多樣化（search result diversification）研究有理論聯繫，可借鑑 xQuAD、PM-2 等經典方法

## 13. ProRAG: Process-Supervised RL for RAG (arXiv: 2601.21912)

### 論文標題
ProRAG: Process-Supervised RL for RAG

### 中文摘要
ProRAG 提出了基於過程監督強化學習的 RAG 優化方法。利用蒙特卡洛樹搜索（MCTS）為多跳檢索的每個步驟生成步驟級獎勵信號，而非僅依賴最終結果的獎勵。這種細粒度的過程監督使模型能夠學習在每個檢索步驟做出最優決策，顯著提升了多跳檢索的效果。

### 核心貢獻
- 將過程監督引入 RAG 的強化學習訓練中
- 使用 MCTS 自動生成步驟級獎勵，無需人工標註中間步驟
- 解決了多跳 RAG 中獎勵稀疏的問題
- 在步驟決策品質和最終回答準確率上均有顯著提升
- 提出了適用於 RAG 的 MCTS 變體設計

### 方法論
1. **問題建模**：將多跳 RAG 建模為序列決策問題
   - 狀態：當前已檢索的資訊和推理進度
   - 動作：下一步的檢索查詢或生成決策
   - 獎勵：最終答案的正確性
2. **MCTS 搜索**：
   - 從當前狀態展開多條可能的檢索路徑
   - 模擬每條路徑到終局，評估結果品質
   - 回溯更新每個中間節點的價值估計
3. **步驟級獎勵生成**：
   - 利用 MCTS 的節點價值作為步驟級獎勵
   - 獎勵反映該步驟對最終成功的貢獻
4. **策略優化**：
   - 基於步驟級獎勵進行策略梯度更新
   - 模型學習在每個決策點選擇最有前途的方向

### 主要發現
- 過程監督比結果監督在多跳問答上提升 15-20%
- MCTS 生成的步驟獎勵與人工標註的相關性達到 0.85+
- 模型學會了更好的檢索查詢策略，減少了無效檢索
- 在 3-5 跳的問題上改善最大，1-2 跳問題改善有限
- 訓練時的 MCTS 搜索開銷在推理時不存在，推理效率不受影響

**研究限制:**
- MCTS 訓練階段的計算成本極高，每個訓練樣本需要大量的模擬展開，限制了可訓練的數據規模
- 對於 1-2 跳的簡單問題改善有限，說明方法的收益集中在特定難度範圍
- MCTS 的搜索品質依賴價值函數的準確性，在新領域中價值函數可能需要重新校準

**其他資訊:**
- 與 AlphaGo/AlphaZero 的 MCTS 訓練範式一脈相承，是將遊戲 AI 的搜索方法遷移到 NLP 任務的成功案例
- 開放問題：是否可以用更輕量的過程監督方法（如基於啟發式的步驟獎勵）達到類似效果，降低訓練成本

## 14. E-mem: Multi-agent Episodic Context Reconstruction (arXiv: 2601.21714)

### 論文標題
E-mem: Multi-agent Episodic Context Reconstruction

### 中文摘要
E-mem 提出了多代理情節記憶重建框架，透過助理代理維護未壓縮的完整記憶，主代理負責協調規劃。這種設計解決了長期對話中上下文壓縮導致的資訊遺失問題，助理代理作為「記憶守護者」保留完整的互動歷史，主代理按需從助理處重建所需的情節上下文。

### 核心貢獻
- 提出主-助理代理架構用於情節記憶管理
- 助理代理維護未壓縮記憶，解決上下文壓縮的資訊損失問題
- 主代理負責高層規劃和記憶需求的識別
- 設計了按需情節重建機制，在需要時精確重建相關上下文
- 在長期對話一致性和記憶準確性上顯著優於壓縮方法

### 方法論
1. **多代理架構**：
   - 主代理（Master Agent）：負責任務規劃、對話管理、決定何時需要歷史記憶
   - 助理代理（Assistant Agents）：各自負責維護特定類型或時段的未壓縮記憶
2. **記憶分配策略**：
   - 不同助理代理負責不同的記憶分區（按時間、主題或交互類型）
   - 每個助理保留其負責範圍內的完整上下文
3. **情節重建機制**：
   - 主代理識別當前任務需要的歷史上下文
   - 向相關助理代理發出記憶查詢請求
   - 助理代理從未壓縮記憶中提取和組織相關情節
   - 重建的情節上下文返回給主代理用於推理
4. **協調與規劃**：
   - 主代理維護全局任務狀態
   - 根據任務需求動態調配助理代理的資源

### 主要發現
- 未壓縮記憶在事實回憶準確率上比壓縮方法高 25-40%
- 多助理分區策略有效解決了單一上下文窗口的容量限制
- 情節重建的精確性取決於主代理識別記憶需求的能力
- 在需要跨時間段整合資訊的任務中優勢最明顯
- 計算成本主要來自助理代理的記憶存儲，可通過分級存儲優化

**研究限制:**
- 維護未壓縮記憶的存儲和計算成本隨對話長度線性增長，在超長期互動中可擴展性受限
- 多代理架構引入了額外的通訊延遲和協調複雜度，對即時對話系統的回應速度有影響
- 記憶分區策略的設計較為靜態，對於主題高度交織的對話可能導致跨分區查詢效率低下

**其他資訊:**
- 與 MemGPT 的分層記憶管理思想有相似性，但採用多代理而非單代理的實現方式
- 開放問題：當助理代理數量增多時，主代理如何高效定位正確的助理，是否需要元索引機制
