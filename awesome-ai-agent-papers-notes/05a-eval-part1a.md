## 1. ClawBench: Evaluating LLM-Based Browser Agents on Live Websites

**論文標題：** ClawBench: Evaluating LLM-Based Browser Agents on Live Websites

**arXiv ID：** 2604.08523

**中文摘要：**
ClawBench 提出一個在真實活躍網站上評估瀏覽器代理的基準測試，涵蓋 163 個真實網站上的 283 個任務。該研究引入「提交攔截」(submission-interception) 機制，在代理即將提交表單或完成交易前攔截動作，藉此在不實際影響真實網站的情況下驗證代理是否正確完成任務。此方法解決了過往基準測試依賴靜態快照或模擬環境的局限性。

**核心貢獻：**
- 建立涵蓋 163 個真實活躍網站、283 個多樣化任務的大規模評估基準
- 提出提交攔截 (submission-interception) 評估方法，避免對真實網站造成副作用
- 支援跨多種任務類型（搜尋、導航、表單填寫、電子商務等）的端到端評估
- 提供對現有瀏覽器代理在真實動態網頁環境中表現的系統性分析

**方法論：**
採用真實網站環境而非模擬器，設計提交攔截層 (submission-interception layer) 在代理執行最終提交動作前擷取其操作意圖。透過比對攔截到的提交內容與預期正確答案來評估任務完成度。任務涵蓋多步驟瀏覽、資訊擷取、表單操作等類別。

**主要發現：**
- 現有瀏覽器代理在真實網站上的表現遠低於在靜態或模擬環境中的表現
- 動態網頁內容、驗證碼、彈出視窗等真實世界因素顯著影響代理表現
- 多步驟任務的成功率隨步驟數增加而急劇下降
- 提交攔截機制有效地在不干擾真實服務的情況下實現了準確評估

**研究限制：**
- 真實網站的動態特性導致評估結果難以精確重現，不同時間點執行可能因網站改版或內容變更而產生不同結果
- 163 個網站雖然數量不少，但仍無法完全代表網際網路的多樣性，尤其在多語言和區域性網站方面可能覆蓋不足
- 提交攔截機制僅適用於有明確提交動作的任務，對於需要持續互動或無明確終點的任務類型評估能力有限

**其他資訊：**
- 與 WebArena、Mind2Web 等先前基準測試形成互補，提供從靜態到動態環境的評估光譜，實務上可結合多個基準進行全面評估
- 提交攔截的設計思路可延伸至其他需要在真實環境中安全評估代理行為的場景，如自動化客服或金融交易代理

## 2. From Features to Actions: Explainability in Agentic AI

**論文標題：** From Features to Actions: Explainability in Agentic AI

**arXiv ID：** 2602.06841

**中文摘要：**
本文探討代理型 AI 系統中的可解釋性問題，比較基於歸因 (attribution-based) 與基於軌跡 (trace-based) 的診斷方法。研究指出傳統的特徵歸因方法在解釋多步驟代理行為時存在根本性不足，需要新的以動作序列為中心的解釋框架來理解代理決策過程。

**核心貢獻：**
- 系統性比較歸因式 (attribution) 和軌跡式 (trace-based) 兩類可解釋性方法在代理系統中的適用性
- 指出從「特徵到動作」的典範轉移：代理 AI 的可解釋性需從輸入特徵歸因轉向動作序列分析
- 提出適用於代理系統的診斷框架，整合因果推理與執行軌跡分析
- 討論可解釋性在代理安全性、除錯和信任建立中的角色

**方法論：**
分析現有可解釋性技術（如 SHAP、LIME、注意力視覺化）應用於代理系統時的侷限，並提出以軌跡為基礎的替代方案。透過案例研究展示如何追蹤代理在多步驟任務中的決策路徑，識別關鍵決策節點，並利用反事實分析理解行為成因。

**主要發現：**
- 傳統特徵歸因方法無法充分解釋代理的多步驟決策行為
- 軌跡式診斷能更好地揭示代理失敗模式和決策瓶頸
- 代理可解釋性需同時考慮單步決策品質和整體策略連貫性
- 結合歸因和軌跡方法的混合式框架具有最佳的診斷效能

**研究限制：**
- 論文以概念性分析和案例研究為主，缺乏大規模實證驗證，所提框架的實際效用尚待更廣泛的量化評估
- 軌跡式診斷的計算和儲存開銷隨代理行為複雜度成指數增長，在大規模部署場景中可能面臨可擴展性瓶頸
- 所討論的可解釋性方法主要針對離散動作空間的代理，對連續控制或混合動作空間的適用性未充分探討

**其他資訊：**
- 與因果推理 (causal reasoning) 和程式除錯 (debugging) 領域有密切關聯，軌跡式分析可借鑑軟體工程中的執行追蹤技術
- 對代理系統的安全審計和監管合規具有直接應用價值，尤其在需要解釋代理決策理由的高風險應用場景中

## 3. Agentic Uncertainty Quantification: LLM Agents Reveal Overconfidence

**論文標題：** Agentic Uncertainty Quantification: LLM Agents Reveal Overconfidence

**arXiv ID：** 2602.06948

**中文摘要：**
本文研究 LLM 代理預測自身任務成功率的能力，揭示代理系統普遍存在的過度自信問題。研究發現代理在執行任務前對成功機率的自我預測顯著高於實際完成率，這種校準偏差 (calibration bias) 在複雜多步驟任務中更為嚴重。

**核心貢獻：**
- 首次系統性研究代理型 AI 系統的不確定性量化與自信度校準
- 揭示 LLM 代理普遍存在的過度自信 (overconfidence) 現象
- 量化代理自我評估與實際表現之間的差距
- 探討改善代理校準度的潛在策略

**方法論：**
設計實驗讓 LLM 代理在執行任務前預測自身的成功機率，然後比較這些預測與實際任務完成結果。跨多個任務領域和難度等級測量校準曲線 (calibration curves)，計算期望校準誤差 (Expected Calibration Error, ECE) 和可靠性圖 (reliability diagrams)。

**主要發現：**
- LLM 代理系統性地高估自身完成任務的能力
- 過度自信在高難度任務和多步驟推理中最為顯著
- 代理的不確定性表達與實際失敗模式之間存在系統性脫節
- 提示工程和自我反思機制可部分改善但無法完全消除過度自信問題

**研究限制：**
- 校準度測量高度依賴任務集合的選擇和難度分佈，不同任務域的結果可能差異顯著，泛化性存疑
- 代理的自信度表達受提示格式影響，不同引導方式可能得到截然不同的校準結果，方法的穩健性有待驗證
- 研究主要關注任務層級的二元成敗判斷，未深入探討部分完成或漸進式成功情況下的不確定性量化

**其他資訊：**
- 與貝氏深度學習和模型校準領域的研究高度相關，可借鑑溫度縮放 (temperature scaling) 等後處理校準技術
- 過度自信的發現對自動化決策系統的部署具有重要警示意義，特別是在醫療或金融等需要準確風險評估的場景中

## 4. AIRS-Bench: Evaluating Frontier AI Research Science Agents

**論文標題：** AIRS-Bench: Automated AI Research Science Benchmark

**arXiv ID：** 2602.06855

**中文摘要：**
AIRS-Bench 提出一個評估 AI 研究科學代理的基準測試，包含從機器學習論文中衍生的 20 個研究任務。這些任務要求代理執行完整的研究流程，包括文獻理解、假設形成、實驗設計、程式碼實現和結果分析，藉此評估代理進行前沿 AI 研究的能力。

**核心貢獻：**
- 建立評估 AI 科學研究代理能力的系統性基準，涵蓋 20 個源自真實 ML 論文的研究任務
- 定義研究代理的多維度能力框架：文獻理解、實驗複現、方法創新
- 提供端到端研究任務評估（從問題理解到實驗執行）
- 分析現有 LLM 在自主科學研究中的能力與限制

**方法論：**
從近期機器學習頂會論文中提取研究問題，將其轉化為可由代理獨立完成的任務。每個任務包含明確的問題描述、所需資源和評估標準。代理需要閱讀相關論文、理解方法、編寫實驗程式碼並產出結果。透過與原論文結果比較及專家評審來評估完成品質。

**主要發現：**
- 現有前沿 LLM 在完整研究流程中仍面臨重大挑戰
- 代理在文獻理解和程式碼生成方面表現較好，但在實驗設計和創新性方面仍有不足
- 多步驟研究流程中的錯誤累積是主要瓶頸
- 需要更好的工具使用和長期規劃能力來支持自主研究

**研究限制：**
- 僅包含 20 個研究任務，樣本量有限且集中於機器學習子領域，對其他科學領域（如生物、物理）的代表性不足
- 評估依賴與原論文結果的比較，但科學研究的正確路徑往往不唯一，限制了對創新性方法的公正評估
- 研究任務的計算成本高昂，完整評估一個代理需要大量 GPU 資源和 API 呼叫費用，限制了可測試的代理數量

**其他資訊：**
- 與 MLAgentBench、SWE-bench 等研究型代理基準形成互補，共同描繪 AI 科學研究能力的完整圖景
- 此基準的結果對「AI 科學家」願景的可行性評估具有參考價值，揭示了從輔助工具到自主研究者的距離仍然相當遙遠

## 5. JADE: Joint Expert-Grounded Dynamic Evaluation

**論文標題：** JADE: Expert-Grounded Dynamic Evaluation via Claim Decomposition

**arXiv ID：** 2602.06486

**中文摘要：**
JADE 提出一種專家基準的動態評估方法，將模型輸出分解為獨立的聲明 (claims)，再逐一對每個聲明進行事實性和品質評估。此方法克服了傳統整體評分的粗糙性，實現更精細、可解釋的評估，同時透過專家標註建立可靠的參考標準。

**核心貢獻：**
- 提出將模型輸出分解為原子聲明 (atomic claims) 的評估框架
- 實現比整體評分更精細和可解釋的品質評估
- 建立專家標註的動態評估基準，避免靜態基準的過時問題
- 展示聲明層級評估如何揭示模型特定的優勢和弱點模式

**方法論：**
首先將模型生成的文本分解為獨立、可驗證的原子聲明。接著由專家對每個聲明標註其正確性、相關性和完整性。評估系統自動執行分解和初步驗證，專家負責最終判定。透過動態更新評估集來防止基準污染 (benchmark contamination)。

**主要發現：**
- 聲明分解揭示了整體評分無法捕捉的細微品質差異
- 不同模型在事實準確性、推理連貫性和資訊完整性上表現各異
- 動態評估能有效抵抗訓練集污染問題
- 專家基準相比自動指標提供更可靠的品質判定

**研究限制：**
- 聲明分解本身可能引入偏差，原子聲明的粒度選擇影響評估結果，且分解過程的一致性難以完全保證
- 依賴專家標註的成本高且擴展性有限，動態更新評估集需要持續投入專家資源
- 對高度主觀或創意性任務（如詩歌寫作、開放式建議）的適用性有限，因為這類輸出難以分解為可客觀驗證的原子聲明

**其他資訊：**
- 與 FActScore 和 SAFE 等基於聲明的事實性評估方法有直接關聯，可視為此類方法向更廣泛品質評估的延伸
- 聲明分解的思路可應用於 RAG 系統的檢索品質評估和幻覺偵測，具有廣泛的實務應用潛力

## 6. Completing Missing Annotation via Multi-Agent Debate

**論文標題：** Completing Missing Annotation via Multi-Agent Debate for Information Retrieval Evaluation

**arXiv ID：** 2602.06526

**中文摘要：**
本文提出利用多代理辯論 (multi-agent debate) 機制來填補資訊檢索 (IR) 評估中的缺失標註。透過讓多個 LLM 代理對未標註文件的相關性進行辯論和投票，生成高品質的偽標籤來補全不完整的相關性判斷集合 (qrels)，從而改善 IR 系統評估的可靠性。

**核心貢獻：**
- 提出多代理辯論框架來解決 IR 評估中缺失標註問題
- 利用代理間的協商和分歧來提高標註品質
- 減少不完整判斷集合對 IR 系統排名評估的偏差
- 展示多代理方法相比單一 LLM 標註的品質提升

**方法論：**
部署多個 LLM 代理，各自獨立評估查詢-文件對的相關性。代理之間進行多輪辯論，交換論點和證據。最終透過投票或共識機制決定相關性標籤。將產生的標籤整合入現有 qrels 中，重新評估 IR 系統排名。與人工標註的金標準比較以驗證方法效度。

**主要發現：**
- 多代理辯論產生的標註品質顯著優於單一代理直接標註
- 辯論過程中的分歧程度可作為標註不確定性的指標
- 補全後的判斷集合使 IR 系統評估更加穩定和可靠
- 該方法在成本效益上優於完全人工標註，同時保持可接受的品質水準

**研究限制：**
- 多代理辯論的 API 呼叫成本隨辯論輪數和代理數量線性增長，大規模應用時的計算成本可能抵消人工標註的節省
- LLM 代理可能共享相似的訓練偏差，導致辯論結果呈現系統性偏向，無法像真正多元的人類標註者那樣提供獨立觀點
- 方法在高度專業或領域特定的查詢（如醫學文獻檢索）上的有效性未充分驗證，可能需要領域適應

**其他資訊：**
- 與 LLM-as-Judge 和多代理協作的研究潮流相呼應，辯論機制在資料標註之外也可應用於事實查核和品質控制
- 對 TREC 等資訊檢索評測活動具有直接影響，可能改變未來相關性評估的標準流程

## 7. TrajAD: Trajectory-Based Anomaly Detection for LLM Agents

**論文標題：** TrajAD: Trajectory-Based Anomaly Detection for LLM Agent Runtime Errors

**arXiv ID：** 2602.06443

**中文摘要：**
TrajAD 提出一種基於執行軌跡的異常偵測方法，用於在運行時識別 LLM 代理的錯誤行為。透過分析代理的動作序列和狀態轉換模式，系統能夠在錯誤擴散之前及早偵測異常軌跡，實現對代理運行時故障的即時監控和預警。

**核心貢獻：**
- 提出針對 LLM 代理執行軌跡的運行時異常偵測框架
- 定義代理軌跡中的正常和異常模式分類學
- 實現即時錯誤偵測，減少錯誤在多步驟執行中的累積
- 建立代理軌跡異常偵測的評估基準

**方法論：**
收集正常和異常代理執行軌跡作為訓練資料。對軌跡中的動作序列、狀態變化和工具調用模式進行建模。利用序列異常偵測技術（如基於距離的方法或學習型偵測器）識別偏離正常模式的軌跡。在不同類型的運行時錯誤上評估偵測的精確率和召回率。

**主要發現：**
- 代理運行時錯誤在軌跡模式中具有可辨識的特徵
- 早期異常偵測可顯著減少因錯誤累積導致的資源浪費
- 不同類型的錯誤（邏輯錯誤、工具使用錯誤、規劃失敗）呈現不同的軌跡特徵
- 結合動作語義和時間模式的偵測方法效果最佳

**研究限制：**
- 異常偵測模型需要大量正常軌跡作為訓練資料，在新部署或任務變更時可能面臨冷啟動問題
- 「正常」行為的定義隨任務和環境變化，固定的異常偵測模型可能在環境漂移 (distribution shift) 時產生大量誤報
- 對於創造性任務或探索性行為，偏離常見模式不一定代表錯誤，可能導致對合理但罕見行為的誤判

**其他資訊：**
- 與軟體系統的運行時監控和可觀測性 (observability) 領域有自然的交叉，可借鑑 APM 工具的設計經驗
- 對生產環境中代理系統的可靠性保障具有直接應用價值，可作為自動重試或人機接管決策的觸發機制

## 8. Emulating Human Choice Behavior with GPT-Based Agents

**論文標題：** Emulating Human Choice Behavior with GPT-Based Agents

**arXiv ID：** 2602.05597

**中文摘要：**
本文研究 GPT 基礎代理重現人類認知偏差和決策行為的能力。實驗涵蓋多種經典的行為經濟學情境，測試代理是否展現與人類相似的認知偏差（如框架效應、錨定效應、損失趨避等），探討 LLM 代理作為人類行為模擬器的可行性和限制。

**核心貢獻：**
- 系統性評估 LLM 代理重現人類認知偏差的能力
- 涵蓋多種經典行為經濟學實驗範式（框架效應、錨定、損失趨避等）
- 分析 LLM 代理行為與人類實驗數據的吻合程度和偏離模式
- 探討 LLM 代理作為社會科學研究中人類行為替代品的可行性

**方法論：**

**論文標題：** Why Do AI Agent Pull Requests Remain Unmerged? A Study of 8,106 Fix-Related PRs

**arXiv ID：** 2602.00164

**中文摘要：**
本文分析 8,106 個 AI 代理提交的修復相關 Pull Requests (PRs)，研究為何許多 AI 生成的 PR 未被合併。研究識別出導致 PR 被拒絕的主要原因類別，包括程式碼品質、上下文理解不足、測試缺失等問題，為改進 AI 程式碼貢獻提供了實證依據。

**核心貢獻：**
- 大規模實證分析 AI 代理 PR 未被合併的原因
- 基於 8,106 個修復相關 PR 的系統性分類研究
- 識別 AI 代理程式碼貢獻的關鍵品質缺陷
- 提出改善 AI 代理 PR 接受率的建議

**方法論：**
從 GitHub 收集 AI 代理（如 Dependabot、Renovate 以及基於 LLM 的工具）提交的修復相關 PR。對未合併的 PR 進行編碼分析，分類拒絕原因。結合審查評論的質性分析和 PR 特徵的量化分析，識別影響合併決策的關鍵因素。

**主要發現：**
- 大量 AI 代理 PR 因缺乏對項目上下文的理解而被拒絕
- 測試不足或測試失敗是常見的拒絕原因
- AI 代理傾向於過度修改或引入不必要的變更
- 與維護者溝通能力的缺乏影響了 PR 的接受
- 較小、聚焦的修改比大範圍重構更容易被接受

**研究限制：**
- 研究涵蓋的 AI 代理類型多元（包括規則式的 Dependabot 和 LLM 式的代理），將其混合分析可能掩蓋不同代理類型的特定問題
- PR 被拒絕的原因分析依賴審查評論的質性編碼，編碼者的主觀判斷可能引入偏差
- 研究僅關注未合併的 PR，未提供與成功合併 PR 的對照分析，難以確定哪些因素是拒絕的充分條件

**其他資訊：**
- 與 Paper 20（AI PR 品質與審查者情感）形成姊妹研究，兩者共同描繪 AI 程式碼貢獻的現狀
- 研究發現對 AI 程式碼助手的產品設計有直接啟示：應強調上下文感知、範圍限制和測試生成，而非僅追求程式碼生成能力

## 17. JAF: Judge Agent Forest for Joint Evaluation

**論文標題：** JAF: Judge Agent Forest for Joint Evaluation Across a Cohort

**arXiv ID：** 2601.22269

**中文摘要：**
JAF (Judge Agent Forest) 提出一種由多個評判代理組成的「森林」架構，用於對一組模型輸出進行聯合評估。不同於逐一獨立評分，JAF 讓評判代理同時考慮整個群組的輸出，透過比較和校準實現更一致、更具區分度的評估結果。

**核心貢獻：**
- 提出跨群組聯合評估的「評判代理森林」框架
- 克服逐一獨立評分導致的尺度不一致和區分度不足問題
- 利用多代理比較機制實現更好的校準
- 展示聯合評估在排名穩定性和人類一致性方面的優勢

**方法論：**
構建多個評判代理（森林中的「樹」），每個代理具有略微不同的評估視角或標準側重。將待評估的一組輸出同時呈現給各評判代理，要求它們在比較中給出相對排序和絕對評分。透過聚合多個評判代理的結果（如投票、平均）產生最終評估。與人類評審結果比較驗證一致性。

**主要發現：**
- 聯合評估比獨立評估產生更穩定和一致的排名
- 多代理聚合有效降低單一評判者的偏差和噪音
- 比較式評估比直接評分更接近人類評審模式
- JAF 在效率和品質之間取得良好平衡

**研究限制：**
- 多代理聚合的計算成本為單一代理的數倍，在大規模評估場景中的效率優勢可能被成本抵消
- 聯合評估需要將整組輸出同時呈現，受限於上下文視窗長度，群組大小存在上限
- 不同「樹」代理的多樣性來源不夠透明，若所有代理基於相同底層模型，多樣性可能僅為表面現象

**其他資訊：**
- 與集成學習 (ensemble learning) 和社會選擇理論 (social choice theory) 有方法論上的共通之處，可借鑑這些領域的聚合策略
- 對 LLM 排行榜 (leaderboard) 的評估方法論具有改進啟示，聯合比較可能比獨立評分產生更可信的排名

## 18. Stalled, Biased, Confused: Understanding Cloud RCA Agent Failures

**論文標題：** Stalled, Biased, Confused: Understanding LLM Agent Failures in Cloud Root Cause Analysis

**arXiv ID：** 2601.22208

**中文摘要：**
本文系統性分析 LLM 代理在雲端根因分析 (Root Cause Analysis, RCA) 中的失敗模式。基於 48,000 個故障場景，研究識別出 16 種失敗類型，包括停滯 (stalled)、偏見 (biased) 和困惑 (confused) 三大類，揭示代理在複雜分散式系統診斷中的根本性挑戰。

**核心貢獻：**
- 基於 48K 故障場景的大規模 RCA 代理失敗分析
- 識別並分類 16 種代理失敗類型
- 歸納三大失敗類別：停滯（卡住不前進）、偏見（先入為主的診斷）、困惑（誤解系統狀態）
- 提出針對各失敗類型的改善策略

**方法論：**
在模擬的雲端環境中注入各種故障（硬體故障、配置錯誤、性能退化等），讓 LLM 代理進行根因分析。系統記錄代理的診斷過程、工具使用和推理軌跡。對失敗案例進行人工分析和編碼，建立失敗類型分類學。統計各類失敗的頻率和影響因素。

**主要發現：**
- 代理最常見的失敗是「停滯」——在診斷路徑上原地踏步
- 「偏見」型失敗中，代理過早鎖定某個假設而忽略矛盾證據
- 「困惑」型失敗源於對分散式系統因果關係的錯誤理解
- 複雜的級聯故障 (cascading failures) 對代理特別困難
- 改善工具使用策略和結構化推理能部分緩解這些問題

**研究限制：**
- 模擬的雲端環境可能無法完全再現生產環境的複雜性，如多租戶干擾、遺留系統互動等真實世界因素
- 故障分類學為人工建構，16 種失敗類型之間的邊界可能存在重疊，編碼一致性需要驗證
- 研究聚焦於失敗案例分析，未提供代理成功診斷案例的對比特徵，限制了改進策略的精確性

**其他資訊：**
- 與 Site Reliability Engineering (SRE) 和 AIOps 領域的研究高度相關，代理化 RCA 是實現自動化運維的關鍵步驟
- 三大失敗類別（停滯、偏見、困惑）的分類對其他代理應用場景也具有參考價值，可能代表 LLM 代理的普遍性弱點

## 19. CAR-bench: Consistency and Limit-Awareness Reasoning Benchmark

**論文標題：** CAR-bench: Evaluating LLM Consistency and Limit-Awareness in Reasoning

**arXiv ID：** 2601.22027

**中文摘要：**
CAR-bench 評估 LLM 的一致性和能力極限意識，測試模型是否能在面對超出能力範圍的問題時適當表達不確定性。基準包含測試回答一致性（同一問題多次提問）和極限感知（應該拒絕回答時是否正確拒絕）的任務，研究模型的自我認知能力。

**核心貢獻：**
- 建立評估 LLM 一致性和能力極限意識的專門基準
- 區分測量回答一致性（穩定性）和極限感知（知道何時不知道）
- 系統性評估模型的不確定性表達和拒答能力
- 分析一致性和準確性之間的關係

**方法論：**
設計兩類測試：(1) 一致性測試——將相同問題以不同措辭或在不同時間多次提出，測量回答的穩定性；(2) 極限感知測試——設計超出模型能力的問題（如需要即時資訊、專業計算或私密知識的問題），評估模型是否能正確識別並表達自身限制。結合定量指標和質性分析。

**主要發現：**
- 多數 LLM 的回答一致性隨問題複雜度增加而顯著下降
- 模型在表達不確定性方面表現不佳，傾向於提供看似自信但錯誤的答案
- 極限感知能力與模型規模不成正比——更大的模型不一定更善於拒答
- 一致性高的回答更可能是正確的，暗示一致性可作為品質信號

**研究限制：**
- 一致性測量受採樣溫度和解碼策略影響顯著，不同推理設定下的結果可能差異很大
- 「超出能力範圍」的問題定義具有時效性——模型更新後原本超出能力的問題可能變得可回答
- 研究未區分模型「知道自己不知道」和「無法生成不確定性表達」兩種不同的極限感知失敗模式

**其他資訊：**
- 與 Paper 3（代理過度自信）的發現相互驗證，共同指向 LLM 在自我認知方面的系統性缺陷
- 一致性作為品質信號的發現對實務部署有直接應用：可透過多次採樣和一致性檢查作為低成本的品質篩選機制

## 20. More Code, Less Reuse: AI Pull Request Quality and Reviewer Sentiment

**論文標題：** More Code, Less Reuse: Analyzing AI-Generated Pull Request Quality and Reviewer Sentiment

**arXiv ID：** 2601.21276

**中文摘要：**
本文分析 AI 生成的 Pull Requests 在程式碼品質和審查者情感反應方面的表現。研究發現 AI 代理傾向於產生更多新程式碼而非重用現有元件，導致程式碼膨脹和維護負擔增加。審查者對 AI PR 的情感反應呈現出特定的正面和負面模式。

**核心貢獻：**
- 實證分析 AI 生成 PR 的程式碼重用率和品質特徵
- 量化「更多程式碼、更少重用」的現象及其影響
- 分析程式碼審查者對 AI PR 的情感和態度
- 識別影響審查者接受度的品質因素

**方法論：**
收集大量 AI 代理提交的 PR 和同類人類提交的 PR 進行比較分析。測量程式碼指標包括：新增程式碼行數、重用現有函式/模組的頻率、重複程式碼比例等。對審查評論進行情感分析，分類審查者的正面和負面回饋。統計分析品質指標與合併決策的關聯。

**主要發現：**
- AI 代理生成的程式碼顯著傾向於重新實現而非重用現有元件
- 此傾向導致更大的差異 (diff size) 和潛在的維護負擔
- 審查者對 AI PR 的負面情感主要集中於缺乏上下文理解和過度修改
- AI PR 中的正面回饋多與格式一致性和低層級修復品質相關
- 較小、聚焦的 AI PR 獲得更正面的審查者情感

**研究限制：**
- 情感分析的準確性有限，審查評論中的反諷、技術幽默或簡潔回饋可能被誤分類
- 研究無法控制 AI PR 和人類 PR 面對的任務難度和類型是否完全可比
- 程式碼重用率的測量方法可能未充分考慮合理的重新實現場景（如原始元件已過時或不適用）

**其他資訊：**
- 與 Paper 16（AI PR 未合併原因）互為補充，共同提供 AI 程式碼貢獻品質的全面圖景
- 「更多程式碼、更少重用」的發現對 AI 程式碼助手的訓練和提示設計有具體啟示：應明確引導模型優先搜尋和重用現有元件
