## 49. Agentic AI Meets Edge Computing in UAV Swarms

**論文標題**: Agentic AI Meets Edge Computing in UAV Swarms  
**arXiv**: 2601.14437

**中文摘要**: 本文探討將基於大型語言模型(LLM)的智能代理系統與邊緣計算結合，應用於無人機(UAV)群集的三種架構方案。研究分析了在資源受限的邊緣環境中部署 LLM 代理的挑戰，包括延遲、頻寬與計算資源的權衡。

**核心貢獻**:
- 提出三種 LLM-邊緣計算整合架構：完全邊緣、雲端-邊緣混合、分層分散式
- 分析各架構在 UAV 群集場景中的延遲、可靠性和自主性權衡
- 探討邊緣環境下模型壓縮與推理最佳化策略

**方法論**: 系統架構設計與比較分析，針對不同任務複雜度和通訊環境評估各架構的適用性。

**主要發現**: 分層架構在多數 UAV 群集場景中表現最優，能平衡自主決策能力與計算資源限制；完全邊緣方案適合延遲敏感任務，但受限於模型能力。

**研究限制**:
- 缺乏大規模實際部署驗證，分析主要基於理論設計與模擬環境，未考慮真實無線通訊的不確定性
- 模型壓縮後的能力衰減評估不夠充分，特別是在複雜決策任務中壓縮模型是否仍能滿足需求
- 未充分討論安全性與對抗場景，如通訊被干擾或節點被入侵時系統的韌性

**其他資訊**:
- 與聯邦學習(federated learning)在邊緣場景的研究有密切關聯，兩者可結合以實現隱私保護的協作式代理學習
- 隨著邊緣 AI 晶片(如 NVIDIA Jetson 系列)性能提升，完全邊緣方案的可行性將持續改善

## 50. Agentic AI: Architectures, Taxonomies, Evaluation

**論文標題**: Agentic AI: Architectures, Taxonomies, and Evaluation  
**arXiv**: 2601.12560

**中文摘要**: 本文提出一套統一的 Agentic AI 分類體系，將智能代理系統分解為感知(Perception)、大腦(Brain)、規劃(Planning)、行動(Action)、工具使用(Tool Use)和協作(Collaboration)六大模組。同時建立了系統性的評估框架。

**核心貢獻**:
- 建立統一的六模組分類框架：感知、大腦、規劃、行動、工具使用、協作
- 對現有 Agentic AI 系統進行系統性分類
- 提出多維度評估方法論，涵蓋任務完成度、效率與安全性
- 整理各模組的設計模式與最佳實踐

**方法論**: 文獻綜述與分類學構建，通過分析大量現有系統歸納出通用架構模式。

**主要發現**: 現有系統在工具使用與協作模組的成熟度差異最大；大腦模組(LLM核心)是系統瓶頸；評估標準缺乏統一性是該領域的主要挑戰。

**研究限制**:
- 分類框架可能過於靜態，難以捕捉快速演化的系統架構(如推理時工具合成等新興範式)
- 評估框架缺乏大規模實證驗證，未證明所提指標能有效區分系統優劣
- 六模組劃分存在邊界模糊問題，例如工具使用與行動模組在實務中可能難以明確區分

**其他資訊**:
- 可作為後續研究者進入該領域的入門地圖，對系統設計者選擇架構模式具有實用參考價值
- 與 Paper 51 的推理分類互補，兩者結合可形成更完整的 Agentic AI 研究全貌

## 51. Agentic Reasoning for LLMs

**論文標題**: Agentic Reasoning for LLMs  
**arXiv**: 2601.12538

**中文摘要**: 本調查論文從基礎推理(foundational)、自我演化(self-evolving)和集體智慧(collective)三個維度，全面梳理 LLM 智能代理推理能力的研究進展。論文分析了從單步推理到多步動態規劃的演進路徑。

**核心貢獻**:
- 提出三維度分析框架：基礎推理、自我演化推理、集體推理
- 系統整理 Chain-of-Thought、Tree-of-Thought 等推理範式的發展脈絡
- 分析自我反思與自我改進機制如何增強代理推理
- 探討多代理協作推理的新興模式

**方法論**: 系統性文獻調查，按三個維度分類並分析超過百篇相關工作。

**主要發現**: 自我演化推理是突破靜態推理天花板的關鍵方向；集體推理通過辯論和協商機制可超越單一代理能力；基礎推理能力仍是上層能力的根本基礎。

**研究限制**:
- 作為調查論文缺乏原創實驗驗證，所有結論均基於他人工作的二次分析
- 三維度框架可能無法涵蓋所有推理模式，如具身推理(embodied reasoning)和因果推理等方向未被充分討論
- 文獻收錄截止時間限制，可能遺漏該領域快速發展中的最新成果

**其他資訊**:
- 集體推理方向與多代理辯論(multi-agent debate)的最新研究高度相關，後者已在數學推理和事實查核中展示出色表現
- 對實務工作者而言，本文提供了選擇推理策略的決策樹：簡單任務用基礎推理、中等任務用自我演化、複雜任務用集體推理

## 52. POLARIS: Typed Planning for Back-Office Automation

**論文標題**: POLARIS: Typed Planning for Back-Office Automation  
**arXiv**: 2601.11816

**中文摘要**: POLARIS 提出基於有向無環圖(DAG)的類型化規劃系統，專為後台辦公自動化設計。系統透過驗證器閘門(validator-gated)執行機制確保每一步驟的輸出符合預期型別約束，大幅提高自動化流程的可靠性。

**核心貢獻**:
- 設計 DAG 結構的任務規劃表示，支援並行與依賴關係
- 引入類型系統約束規劃步驟的輸入輸出
- 實現驗證器閘門執行：每步完成後需通過型別驗證才能繼續
- 針對企業後台流程(如報表生成、資料處理)的實際應用

**方法論**: 結合形式化類型系統與 LLM 規劃能力，在 DAG 執行引擎中嵌入驗證邏輯。

**主要發現**: 類型化規劃相比自由形式規劃顯著降低執行錯誤率；驗證器閘門有效攔截中間步驟錯誤，避免錯誤傳播；DAG 結構比線性規劃更好地利用並行性。

**研究限制**:
- 類型系統的設計需要領域專家參與，前期定義型別約束的人力成本較高，可能限制快速部署
- 僅在後台辦公自動化場景中驗證，對於非結構化或創意性任務的泛化能力未知
- 驗證器閘門增加了執行延遲，在需要即時回應的場景中可能成為瓶頸

**其他資訊**:
- 與程式語言中的漸進式型別(gradual typing)思想相似，可考慮引入可選型別約束以平衡靈活性與安全性
- 企業 RPA(機器人流程自動化)廠商可直接借鑑此架構，為現有工作流引擎增加 LLM 規劃能力

## 53. From Everything-is-a-File to Files-Are-All-You-Need

**論文標題**: From Everything-is-a-File to Files-Are-All-You-Need  
**arXiv**: 2601.11672

**中文摘要**: 本文借鑑 Unix「一切皆檔案」的設計哲學，提出「檔案即所需」的 Agentic AI 設計原則。主張以檔案系統作為代理間通訊、狀態管理和工具介面的統一抽象層，簡化系統複雜度。

**核心貢獻**:
- 將 Unix 哲學遷移到 AI 代理系統設計
- 提出以檔案為中心的代理通訊與狀態管理機制
- 論證簡單介面(純文字、結構化檔案)對可組合性的重要性
- 降低代理系統的耦合度與認知複雜度

**方法論**: 設計原則論證，結合 Unix 系統設計經驗與現代 AI 代理需求進行類比推理。

**主要發現**: 檔案抽象提供了天然的持久化、版本控制和存取控制機制；簡單的文字介面比複雜 API 更適合 LLM 代理互動；Unix pipe 模式可直接映射到代理工作流。

**研究限制**:
- 檔案系統抽象在高頻即時通訊場景中可能產生嚴重的 I/O 瓶頸，不適合毫秒級互動需求
- 論證主要基於類比推理缺乏量化實驗，未提供檔案介面與傳統 API 介面的效能對比數據
- 忽略了分散式環境中的檔案一致性問題，多節點同時讀寫可能導致競態條件

**其他資訊**:
- 與 Plan 9 作業系統的設計理念一脈相承，研究者可從 Plan 9 的分散式檔案系統設計中獲取靈感
- 對 MCP(Model Context Protocol)等代理通訊協議的設計具有啟發意義，提示簡單性應優先於功能豐富性

## 54. Towards AGI: Self Evolving Agent

**論文標題**: Towards AGI: Self Evolving Agent  
**arXiv**: 2601.11658

**中文摘要**: 本文提出一種階層式自我演化代理架構，結合課程學習(curriculum learning)與遺傳演算法(GA)，使代理能在無需人工干預的情況下持續提升自身能力。系統通過由簡入繁的任務序列引導能力成長。

**核心貢獻**:
- 設計階層式自我演化架構，支援多層次能力提升
- 整合課程學習策略，自動生成由易到難的訓練任務序列
- 採用遺傳演算法對代理策略進行選擇與變異
- 實現無需外部監督的自主能力演進

**方法論**: 結合課程學習的漸進式難度調節與遺傳演算法的族群演化，在階層式架構中實現自動化能力迭代。

**主要發現**: 課程學習有效避免了直接面對困難任務的學習失敗；遺傳演算法在策略空間探索中表現優於純梯度方法；階層式設計允許不同層次獨立演化。

**研究限制**:
- 遺傳演算法的計算成本隨族群規模和演化代數急劇增長，大規模應用的可擴展性存疑
- 自動生成的課程任務品質缺乏保證，錯誤的難度排序可能導致學習路徑偏差
- 實驗僅在有限的任務域中驗證，是否能演化出通用智能(AGI)能力缺乏理論支撐

**其他資訊**:
- 與開放式學習(open-ended learning)和 POET(Paired Open-Ended Trailblazer)等自動課程生成研究高度相關
- 提出了一個重要的開放問題：自我演化系統是否會陷入局部最優而無法突破，需要何種機制確保持續探索

## 55. EvoFSM: Self-Evolution with Finite State Machines

**論文標題**: EvoFSM: Self-Evolution with Finite State Machines  
**arXiv**: 2601.09465

**中文摘要**: EvoFSM 提出以有限狀態機(FSM)作為代理行為的顯式表示，取代傳統的自由形式程式碼生成。透過演化機制自動修改和擴展 FSM 的狀態與轉移規則，實現可解釋且可控的代理自我演化。

**核心貢獻**:
- 以 FSM 作為代理行為的結構化表示，提供顯式可解釋性
- 設計針對 FSM 結構的演化運算子（新增狀態、修改轉移、合併狀態）
- 相比自由程式碼生成更容易驗證和除錯
- 演化過程中保持行為的結構一致性

**方法論**: 將代理行為編碼為 FSM，利用演化演算法對 FSM 結構進行搜索和最佳化，每代通過任務評估進行選擇。

**主要發現**: FSM 表示在可解釋性和可驗證性上顯著優於自由程式碼；演化產生的 FSM 結構趨於簡潔且高效；結構化約束加速了有意義行為的發現。

**研究限制**:
- FSM 的表達能力本質上有限，無法表示需要記憶或計數的複雜行為(如需要堆疊或遞迴的任務)
- 隨著狀態數增加，FSM 的可解釋性優勢會迅速減弱，大型 FSM 同樣難以理解
- 演化搜索在高維狀態空間中效率下降，狀態與轉移數量的組合爆炸是根本性挑戰

**其他資訊**:
- 可與層級式 FSM 或行為樹(Behavior Tree)結合以提升表達能力，遊戲 AI 領域的相關經驗值得借鑑
- 對需要可審計性的安全關鍵應用(如自動駕駛決策模組)具有實際應用價值

## 56. Tool-Memory Conflicts in Tool-Augmented LLMs

**論文標題**: Tool-Memory Conflicts in Tool-Augmented LLMs  
**arXiv**: 2601.09760

**中文摘要**: 本文研究工具增強型 LLM 中內部知識與外部工具輸出之間的衝突問題。當 LLM 的參數化記憶與工具返回結果矛盾時，模型行為表現出不一致性，可能導致錯誤推理或忽略有效工具輸出。

**核心貢獻**:
- 系統性識別和分類工具-記憶衝突的類型與成因
- 建構評估基準測量模型在衝突情境中的行為
- 分析不同衝突解決策略（偏向工具 vs. 偏向記憶）的優劣
- 提出緩解衝突的方法建議

**方法論**: 設計對照實驗，構造工具輸出與模型內部知識矛盾的測試案例，觀察模型的選擇傾向與推理品質。

**主要發現**: LLM 在高信心知識領域傾向忽略工具輸出；衝突導致的推理錯誤率顯著高於無衝突情境；明確的衝突解決指令可部分緩解問題但無法根除。

**研究限制**:
- 構造的衝突測試案例可能過於人工化，與真實使用場景中自然發生的衝突情境有差距
- 僅測試有限數量的 LLM，不同模型架構和訓練方式可能展現不同的衝突處理模式
- 未深入分析衝突的根因(如訓練數據分布與工具知識覆蓋的差異)，停留在現象描述層面

**其他資訊**:
- 與 RAG(檢索增強生成)中的知識衝突問題高度相關，兩者共享相似的信任校準挑戰
- 對工具增強系統的提示工程(prompt engineering)實務有直接指導意義：應在系統提示中明確指定衝突時的優先順序

## 57. MAXS: Meta-Adaptive Exploration

**論文標題**: MAXS: Meta-Adaptive Exploration  
**arXiv**: 2601.09259

**中文摘要**: MAXS 提出元適應探索策略，通過前瞻規劃(lookahead planning)估計工具使用的預期價值，動態決定何時調用工具、調用哪個工具。系統能根據任務狀態自適應地平衡探索與利用。

**核心貢獻**:
- 設計前瞻規劃機制評估工具調用的預期收益
- 實現元層級的自適應探索策略
- 動態工具選擇：根據預期資訊增益決定工具調用順序
- 減少不必要的工具調用，提高效率

**方法論**: 利用前瞻搜索模擬不同工具調用路徑的結果，通過價值估計函數評選最佳行動序列。

**主要發現**: 前瞻規劃顯著減少冗餘工具調用；自適應策略在不同任務複雜度下均優於固定策略；工具價值估計的準確度是系統性能的關鍵因素。

**研究限制**:
- 前瞻搜索的計算開銷隨搜索深度呈指數增長，在工具數量眾多或序列較長時可能不可行
- 價值估計函數的準確度依賴於環境模型的品質，在不確定性高的場景中估計偏差可能導致次優決策
- 假設工具調用結果可被合理模擬，但許多真實工具(如網路搜索)的輸出難以預測

**其他資訊**:
- 與蒙地卡羅樹搜索(MCTS)在決策規劃中的應用思路一致，可借鑑 AlphaGo 的搜索策略改進前瞻效率
- 對 API 成本敏感的應用場景特別有價值，減少不必要的工具調用可直接降低運營成本

## 58. ToolACE-MCP: History-Aware Routing

**論文標題**: ToolACE-MCP: History-Aware Routing  
**arXiv**: 2601.08276

**中文摘要**: ToolACE-MCP 為 Model Context Protocol (MCP) 生態系統設計了歷史感知路由器，能根據過往工具使用模式和對話歷史智能地將請求路由至最適當的工具服務。系統有效解決了大規模 MCP 工具生態中的工具選擇問題。

**核心貢獻**:
- 設計歷史感知的工具路由機制，利用使用模式提升選擇準確度
- 針對 MCP 協議生態系統的專門最佳化
- 支援動態工具註冊與去註冊
- 路由決策融合語義匹配與歷史統計

**方法論**: 結合對話上下文編碼與歷史使用模式統計，訓練路由模型在大規模工具集中進行高效選擇。

**主要發現**: 歷史感知路由相比純語義匹配提升工具選擇準確率；隨著工具生態規模增長，路由器的價值越發顯著；使用模式包含豐富的工具相關性資訊。

**研究限制**:
- 歷史模式偏差可能導致冷啟動問題，新上線的工具因缺乏使用歷史而被系統性地低估
- MCP 協議仍處於早期階段，研究結果的持久性取決於協議規範的穩定性
- 路由模型的訓練數據可能存在偏見，過度依賴歷史模式可能錯過更優的新工具選擇

**其他資訊**:

**方法論**: 建立執行-評估-學習的閉環：執行工具調用，評估結果品質，將成功/失敗案例轉化為訓練信號。

**主要發現**: 數據飛輪機制使工具使用能力持續提升而非靜態固定；自動生成的訓練數據品質可比人工標註；校準後模型有效減少了不必要的工具調用。

**研究限制**:
- 自我生成的訓練數據可能包含系統性偏差，長期積累可能導致能力退化(reward hacking)
- 評估模組本身的品質是飛輪有效性的前提，錯誤的評估會產生錯誤的訓練信號
- 飛輪的冷啟動階段需要足夠品質的初始數據，完全零起步的場景可能效果不佳

**其他資訊**:
- 與自我訓練(self-training)和半監督學習的思路一致，但增加了工具互動的外部反饋信號使學習更可靠
- 對需要長期運行的代理服務(如客服機器人、編程助手)具有高度實用性，系統可隨使用時間自動改進

## 70. No More Stale Feedback: Co-Evolving Critics

**論文標題**: No More Stale Feedback: Co-Evolving Critics  
**arXiv**: 2601.06794

**中文摘要**: 本文提出同步演化的批評者(co-evolving critics)機制，使用同步化的群組相對策略最佳化(GRPO)同時訓練代理和其批評者模型。解決了傳統方法中批評者反饋因代理進步而過時的問題。

**核心貢獻**:
- 識別「陳舊反饋」問題：批評者跟不上代理進步
- 設計同步 GRPO 同時演化代理與批評者
- 保持批評者與代理能力水準的同步
- 提供更準確、更具挑戰性的反饋信號

**方法論**: 在每個訓練迭代中同時更新代理策略和批評者模型，使用相對排序而非絕對評分。

**主要發現**: 同步演化避免了批評者過時導致的訓練停滯；GRPO 的相對排序比絕對評分更穩健；共同演化產生了更強的最終代理。

**研究限制**:
- 同步訓練兩個模型的計算資源需求近乎翻倍，對硬體要求較高
- 共同演化可能導致代理與批評者陷入共振(co-adaptation)，雙方互相適應而非真正提升
- 相對排序在候選回應品質都很低或都很高時區分度下降，極端情況下可能失效

**其他資訊**:
- 與生成對抗網路(GAN)的訓練動態有相似之處，可借鑑 GAN 訓練穩定性的研究成果
- 對 RLHF(人類反饋強化學習)的獎勵模型更新策略有啟示：獎勵模型也應隨策略模型同步演化

## 71. CEDAR: Context Engineering for Data Science

**論文標題**: CEDAR: Context Engineering for Data Science  
**arXiv**: 2601.06606

**中文摘要**: CEDAR 提出針對數據科學任務的上下文工程(context engineering)框架，通過結構化提示(structured prompting)和規劃/程式碼代理分離的設計，提升 LLM 在資料分析和建模任務中的表現。

**核心貢獻**:
- 設計數據科學專用的結構化提示框架
- 分離規劃代理與程式碼執行代理的職責
- 最佳化上下文組織以適應數據科學工作流
- 提供資料理解、分析和視覺化的端到端流程

**方法論**: 將數據科學任務分解為規劃(高層推理)和執行(程式碼生成)兩階段，各由專門代理負責，通過結構化上下文傳遞資訊。

**主要發現**: 分離規劃與執行顯著提升了複雜分析任務的成功率；結構化上下文比自由提示更有效利用有限的上下文窗口；專業分工減少了單一代理的認知負擔。

**研究限制**:
- 結構化提示框架的設計高度依賴數據科學領域知識，遷移至其他領域需要重新設計
- 規劃與執行的分離可能導致資訊損失，規劃代理無法預見所有執行層面的技術限制
- 評估主要在標準數據科學基準上進行，對於探索性分析(EDA)等開放式任務的效果未充分驗證

**其他資訊**:
- 與 AutoML 和自動化數據科學工具(如 Auto-sklearn)的目標互補，前者專注模型選擇，後者專注整體工作流
- 對數據科學團隊的實務工作有直接助益，可作為 Jupyter Notebook 的智能輔助層整合至現有工作流

## 72. ArenaRL: Tournament-based Relative Ranking

**論文標題**: ArenaRL: Tournament-based Relative Ranking  
**arXiv**: 2601.06487

**中文摘要**: ArenaRL 提出以錦標賽制相對排序替代傳統逐點評分的強化學習獎勵機制。通過讓多個代理輸出相互競爭排名，獲得比絕對分數更穩定且更具區分度的訓練信號。

**核心貢獻**:
- 以錦標賽排名替代逐點絕對評分
- 相對排序提供更穩健的獎勵信號
- 減少評分器偏差和校準問題
- 支援多代理同時競爭與排名

**方法論**: 設計錦標賽機制讓多個候選回應相互比較，使用 Elo 或類似排名系統產生相對獎勵信號，用於策略梯度更新。

**主要發現**: 相對排序比絕對評分更不受評分器偏差影響；錦標賽制在區分相近品質輸出時更有效；排名信號的方差低於逐點評分，訓練更穩定。

**研究限制**:
- 錦標賽比較的計算成本隨候選數量平方增長(O(n^2)配對)，大規模候選集的排名效率是挑戰
- Elo 評分系統假設勝率的遞移性(transitivity)，但 LLM 輸出品質的偏好可能存在非遞移性
- 排名信號僅提供序數資訊而非基數資訊，無法區分「略優」和「遠優」的差異程度

**其他資訊**:
- 與 Chatbot Arena 的排名方法論一致，驗證了相對比較在 LLM 評估中的有效性
- 可與 Paper 70 的共同演化批評者結合：使用錦標賽排名作為共同演化的訓練信號
