## 61. RealMem: Real-World Memory-Driven Interaction

**論文標題：** RealMem: Real-World Memory-Driven Interaction (arXiv: 2601.06966)

**中文摘要：**
RealMem 提出一個以記憶驅動的跨對話互動評估基準，包含超過 2000 組跨會話對話，專門用於評估 LLM 代理在多輪互動中對使用者目標演變的追蹤與記憶能力。該基準模擬真實世界中使用者需求隨時間變化的情境，測試模型是否能在長期互動中維持一致性並適應目標更新。

**核心貢獻：**
- 建立首個大規模跨會話記憶驅動互動基準，包含 2000+ 組對話
- 設計動態目標演變機制，模擬真實使用者需求的時間性變化
- 提出記憶一致性與目標追蹤的量化評估指標
- 揭示現有 LLM 在長期記憶維護方面的顯著不足

**方法論：**
- 構建多會話對話場景，每組對話包含多輪互動且使用者目標會隨時間演變
- 設計目標追蹤任務，要求模型記住先前對話中的偏好、約束和決策
- 引入記憶衝突與更新情境，測試模型處理矛盾資訊的能力
- 利用人工標註建立黃金標準答案，衡量記憶準確度與目標對齊程度

**主要發現：**
- 現有 LLM 在跨會話記憶任務上表現顯著下降，尤其當目標演變複雜時
- 模型傾向於遺忘早期偏好或無法識別目標的細微變化
- 記憶增強策略（如外部記憶模組）可顯著改善跨會話表現
- 揭示了記憶保持與目標適應之間的權衡關係

**研究限制：**
- 對話場景主要基於合成資料生成，可能無法完全反映真實使用者互動的複雜性與多樣性
- 目標演變模式相對簡化，真實世界中使用者需求變化可能更為隱晦且非線性
- 評估僅涵蓋英語對話，跨語言記憶能力未被探討

**其他資訊：**
- 該基準為研究個人化長期助手提供了重要的測試基礎設施
- 記憶增強模組的設計可作為未來 LLM 記憶架構改進的參考方向

## 62. IDRBench: Interactive Deep Research Benchmark

**論文標題：** IDRBench: Interactive Deep Research Benchmark (arXiv: 2601.06676)

**中文摘要：**
IDRBench 是首個互動式深度研究基準，內建使用者模擬器，用於評估 LLM 代理在多輪互動研究任務中的表現。該基準要求代理主動提問、收集資訊、整合多源知識，並在與使用者的互動中逐步深化研究結論。

**核心貢獻：**
- 提出首個互動式深度研究評估基準
- 設計使用者模擬器（User Simulator），能根據研究進展提供適當回饋
- 建立多維度評估框架，涵蓋研究深度、互動品質、結論準確性
- 填補現有基準在互動式研究能力評估方面的空白

**方法論：**
- 設計多階段研究任務，要求代理透過與使用者的多輪對話逐步深入
- 開發自動化使用者模擬器，模擬真實使用者的資訊提供模式與回應行為
- 建立研究品質評估指標，包括資訊覆蓋率、邏輯連貫性、結論可靠性
- 比較不同互動策略（主動提問 vs. 被動接收）對研究品質的影響

**主要發現：**
- 主動提問策略顯著優於被動研究模式
- 現有模型在判斷何時該提問、提問什麼方面仍有顯著不足
- 使用者模擬器的品質對評估結果有重要影響
- 互動輪數與研究品質呈正相關，但存在邊際效益遞減

**研究限制：**
- 使用者模擬器的行為模式可能與真實使用者存在差異，影響評估的生態效度
- 研究任務的領域覆蓋有限，未涵蓋高度專業化的科學研究場景
- 評估指標主要依賴自動化度量，對研究品質的深層面向（如洞察力）難以捕捉

**其他資訊：**
- 該基準的互動式設計對未來研究助手產品的評估具有直接應用價值
- 使用者模擬器的開源有助於社群進一步改善互動研究評估方法

## 63. ToolGym: Open-world Tool-using Environment

**論文標題：** ToolGym: Open-world Tool-using Environment (arXiv: 2601.06328)

**中文摘要：**
ToolGym 構建了一個開放世界的工具使用環境，包含跨越 204 個應用程式的 5571 個工具，並引入失敗注入機制來測試代理的錯誤處理能力。該環境模擬真實世界中工具使用的複雜性，包括 API 變動、服務中斷等異常情況。

**核心貢獻：**
- 建立大規模開放世界工具使用環境，涵蓋 5571 個工具、204 個應用程式
- 引入失敗注入（Failure Injection）機制，模擬真實世界 API 故障
- 支援工具組合與多步驟推理的複雜任務
- 提供開放式任務設定，無固定工具組合限制

**方法論：**
- 從真實應用程式中提取工具定義，建立大規模工具庫
- 設計失敗注入框架，包括超時、錯誤回傳、格式變化等多種故障模式
- 構建多難度等級的任務集，從單工具到多工具協作
- 評估模型的工具發現、選擇、呼叫與錯誤恢復能力

**主要發現：**
- 現有 LLM 在面對工具失敗時的恢復能力普遍不足
- 開放世界設定下的工具選擇準確率顯著低於封閉環境
- 失敗注入大幅降低所有模型的任務完成率
- 模型在工具組合推理方面存在顯著瓶頸

**研究限制：**
- 工具定義基於靜態快照，未考慮 API 隨時間演進的版本變化
- 失敗注入模式為預定義類型，可能無法涵蓋所有真實世界故障情境
- 任務難度的定義較為粗糙，缺乏更細粒度的困難度分層

**其他資訊：**
- 該環境可作為訓練代理錯誤恢復能力的強化學習平台
- 開放世界設定對未來通用型工具使用代理的開發具有重要參考意義

## 64. TowerMind: Tower Defence Game for LLM Agents

**論文標題：** TowerMind: Tower Defence Game for LLM Agents (arXiv: 2601.05899)

**中文摘要：**
TowerMind 利用塔防遊戲作為評估框架，測試 LLM 代理的規劃、決策與資源管理能力。塔防遊戲的即時策略特性要求代理在動態環境中做出連續決策，提供了一個直觀且可量化的評估場景。

**核心貢獻：**
- 提出基於塔防遊戲的 LLM 代理評估框架
- 設計多維度評估指標，涵蓋策略規劃、資源配置、適應性決策
- 建立多難度等級的遊戲場景，支援漸進式能力評估
- 揭示 LLM 在即時策略決策中的局限性

**方法論：**
- 實作塔防遊戲環境，包含多種塔類型、敵人波次與地圖配置
- 將遊戲狀態轉換為文字描述，作為 LLM 的輸入
- 設計決策介面，要求模型選擇放置位置、升級策略與資源分配
- 透過遊戲得分、存活波次、資源效率等指標量化表現

**主要發現：**
- LLM 在長期策略規劃方面表現不佳，傾向於短視決策
- 空間推理能力是主要瓶頸，模型難以理解地圖拓撲
- 資源管理決策品質隨遊戲複雜度增加而顯著下降
- 提供遊戲歷史資訊可改善決策品質，但改善幅度有限

**研究限制：**
- 塔防遊戲的決策空間相對受限，可能無法完全代表通用策略規劃能力
- 文字化的遊戲狀態描述可能遺失視覺空間中的關鍵資訊
- 缺乏與人類玩家的直接對比基線，難以量化能力差距的實際意義

**其他資訊：**
- 遊戲化評估的設計降低了評估門檻，便於社群廣泛參與
- 該框架可擴展至其他策略遊戲類型以評估不同面向的決策能力

## 65. MineNPC-Task: Memory-Aware Minecraft Agents

**論文標題：** MineNPC-Task: Memory-Aware Minecraft Agents (arXiv: 2601.05215)

**中文摘要：**
MineNPC-Task 提出一個以 Minecraft 為基礎的記憶感知代理基準，由使用者撰寫的任務範本構成，具有參數化設計，可生成多樣化的測試實例。該基準特別關注代理在開放世界遊戲中利用記憶資訊完成複雜任務的能力。

**核心貢獻：**
- 建立使用者撰寫的參數化任務範本系統
- 設計記憶感知機制，測試代理利用歷史互動資訊的能力
- 在 Minecraft 開放世界環境中提供多樣化的任務場景
- 支援可控的任務難度調整與實例生成

**方法論：**
- 設計參數化任務範本，透過參數替換生成多樣化測試實例
- 建立記憶模組，儲存代理與環境的歷史互動記錄
- 在 Minecraft 環境中部署 NPC 代理，執行建造、探索、合作等任務
- 評估代理的記憶召回、任務規劃與執行能力

**主要發現：**
- 記憶增強顯著改善代理在需要歷史資訊的任務上的表現
- 參數化範本有效提升了評估的多樣性與可重複性
- 代理在需要長期記憶的複雜任務鏈中表現仍不理想
- 記憶資訊的結構化組織對任務成功率有重要影響

**研究限制：**
- Minecraft 環境的模擬延遲與技術複雜性限制了大規模評估的可行性
- 任務範本由少數使用者撰寫，可能存在設計偏差與覆蓋盲區
- 記憶模組的設計為特定架構，不同記憶實現方式的公平比較受限

**其他資訊：**
- 參數化範本設計可推廣至其他開放世界遊戲的代理評估
- 該基準的任務生成機制有助於持續擴展測試集以對抗過擬合

## 66. Internal Representations as Indicators of Hallucinations

**論文標題：** Internal Representations as Indicators of Hallucinations (arXiv: 2601.05214)

**中文摘要：**
本文探索利用 LLM 的內部表示（internal representations）來檢測工具呼叫幻覺（tool-calling hallucinations）。研究發現模型的隱藏層狀態包含關於輸出可靠性的重要訊號，可用於在推理時預測幻覺的發生。

**核心貢獻：**
- 首次系統性研究內部表示與工具呼叫幻覺之間的關係
- 開發基於隱藏層表示的幻覺檢測方法
- 建立工具呼叫幻覺的分類體系與標註資料集
- 提供可在推理時部署的輕量級幻覺預警機制

**方法論：**
- 收集 LLM 在工具呼叫任務中的內部隱藏層表示
- 訓練探針分類器（probe classifier）從內部表示預測幻覺
- 分析不同層級、不同 token 位置的表示對檢測效果的影響
- 比較內部表示方法與基於輸出的幻覺檢測方法

**主要發現：**
- 模型內部表示確實包含可區分正確與幻覺輸出的訊號
- 中間層的表示通常比最後幾層更具預測力
- 工具名稱與參數位置的 token 表示最具資訊量
- 該方法可作為推理時的即時幻覺偵測機制，且計算開銷較低

**研究限制：**
- 需要存取模型內部隱藏層，僅適用於開源或可存取中間狀態的模型，不適用於閉源 API
- 探針分類器的泛化能力受限於訓練資料的分布，面對新型工具或領域可能失效
- 研究僅針對工具呼叫場景，是否適用於其他類型的幻覺（如事實性幻覺）尚未驗證

**其他資訊：**
- 該方法的低計算開銷使其適合部署於生產環境中作為即時監控機制
- 研究成果對理解 LLM 內部「不確定性」的編碼方式有理論貢獻

## 67. Agent-as-a-Judge

**論文標題：** Agent-as-a-Judge (arXiv: 2601.05111)

**中文摘要：**
本文提供從 LLM-as-a-Judge 到 Agent-as-a-Judge 的全面調查，系統性地回顧利用 LLM 代理作為自動評估者的研究進展。文章探討了代理評估器相較於傳統 LLM 評估器的優勢，包括工具使用、多步推理與環境互動能力。

**核心貢獻：**
- 系統性調查從 LLM-as-a-Judge 到 Agent-as-a-Judge 的演進
- 建立 Agent-as-a-Judge 的分類框架與設計原則
- 分析代理評估器在不同應用場景中的優劣勢
- 提出未來研究方向與開放問題

**方法論：**
- 文獻綜述方法，系統性蒐集與分析相關研究
- 建立多維度比較框架，涵蓋能力、可靠性、適用範圍
- 案例分析不同應用場景中的 Agent-as-a-Judge 實踐
- 歸納設計模式與最佳實踐

**主要發現：**
- Agent-as-a-Judge 在需要驗證性判斷的任務中顯著優於 LLM-as-a-Judge
- 工具使用能力使代理評估器可執行程式碼、查詢資料庫以驗證答案
- 多步推理使評估過程更透明、可追溯
- 但代理評估器成本更高，且可能引入新的偏差來源

**研究限制：**
- 作為綜述論文，缺乏統一實驗條件下的原創性實證比較
- 所調查的研究多集中於英語場景，多語言環境下的適用性討論不足
- 對代理評估器自身偏差與失敗模式的分析深度有限

**其他資訊：**
- 該調查為設計代理評估系統的從業者提供了實用的設計模式參考
- 文中提出的開放問題可作為後續研究的路線圖

## 68. Arabic Prompts with English Tools

**論文標題：** Arabic Prompts with English Tools (arXiv: 2601.05101)

**中文摘要：**
本文提出首個阿拉伯語工具呼叫基準，專門評估 LLM 在使用阿拉伯語提示但工具定義為英語的跨語言情境中的工具使用能力。研究揭示了語言不匹配對工具呼叫準確性的顯著影響。

**核心貢獻：**
- 建立首個阿拉伯語工具呼叫評估基準
- 揭示跨語言工具使用中的語言不匹配問題
- 提供阿拉伯語 NLP 在代理能力評估方面的重要資源
- 分析不同 LLM 在跨語言工具呼叫中的表現差異

**方法論：**
- 構建阿拉伯語使用者查詢與英語工具定義的配對資料集
- 設計多層級工具呼叫任務，從簡單到複雜
- 評估模型的意圖理解、參數提取與工具選擇能力
- 對比阿拉伯語查詢與對應英語查詢的表現差異

**主要發現：**
- 所有模型在阿拉伯語查詢下的工具呼叫準確率顯著低於英語
- 參數提取是主要錯誤來源，尤其涉及阿拉伯語數字與日期格式
- 語言不匹配導致意圖理解偏差，增加工具選擇錯誤
- 多語言預訓練程度較高的模型表現相對較好

**研究限制：**
- 僅涵蓋阿拉伯語一種非英語語言，研究結論是否適用於其他語言需進一步驗證
- 工具定義僅使用英語，未探索雙語或翻譯工具定義的潛在改善效果
- 資料集規模相對有限，可能未涵蓋阿拉伯語的所有方言變體

**其他資訊：**
- 該基準填補了非英語工具使用評估的重要空白，可啟發其他語言的類似研究
- 研究結果對開發多語言代理產品的企業具有直接實用價值

## 69. Effects of Personality Steering on Cooperative Behavior

**論文標題：** Effects of Personality Steering on Cooperative Behavior (arXiv: 2601.05302)

**中文摘要：**
本文研究大五人格特質（Big Five Personality）導向對 LLM 在囚徒困境（Prisoner's Dilemma）中合作行為的影響。透過人格提示工程，研究者操控模型展現不同人格特質，並觀察其對合作決策的系統性影響。

**核心貢獻：**
- 系統性研究五大人格特質對 LLM 合作行為的影響
- 在囚徒困境博弈框架中量化人格導向效果
- 揭示不同人格維度對合作傾向的差異化影響
- 為 LLM 行為控制與對齊研究提供實證依據

**方法論：**
- 使用系統提示注入大五人格特質（開放性、盡責性、外向性、親和性、神經質）
- 在重複囚徒困境中測試 LLM 的合作/背叛決策
- 控制變數包括人格特質強度、對手策略、遊戲輪次
- 統計分析人格維度與合作率之間的相關性

**主要發現：**
- 親和性（Agreeableness）對合作率影響最大，高親和性顯著增加合作傾向
- 神經質（Neuroticism）傾向降低合作率，增加不穩定決策
- 人格導向效果在不同模型間存在差異，但整體趨勢一致
- 重複博弈中人格效應隨互動次數增加而趨於穩定

**研究限制：**
- 囚徒困境為高度簡化的互動場景，無法涵蓋現實中更複雜的社會合作情境

**方法論：**
- 定義通用追蹤格式（trace format），記錄代理間通訊、決策與動作
- 設計評估管線，從追蹤中提取可靠性指標
- 支援多種主流多代理框架的追蹤收集
- 建立自動化評估報告生成機制

**主要發現：**
- 統一追蹤格式能有效支援跨框架的公平比較
- 多代理系統的可靠性問題往往源於代理間通訊而非單一代理
- 框架選擇對最終表現的影響小於代理設計與提示品質
- 系統性評估能揭示僅靠最終結果評估無法發現的問題

**研究限制：**
- 統一追蹤格式可能無法捕捉特定框架的獨有特性與優勢
- 可靠性指標的定義偏向通用性，可能遺漏領域特定的品質要求
- 評估套件本身的維護成本隨支援框架數量增加而上升

**其他資訊：**
- 框架無關的設計理念對建立多代理系統的行業標準具有推動意義
- 追蹤格式的標準化有助於多代理系統的除錯與效能優化

## 80. Beyond Perfect APIs: WildAGTEval

**論文標題：** Beyond Perfect APIs: WildAGTEval (arXiv: 2601.00268)

**中文摘要：**
WildAGTEval 評估 LLM 在真實 API 複雜性下的函數呼叫能力，超越了傳統基準中理想化 API 的假設。基準涵蓋不完整文件、版本差異、邊界情況等真實世界中常見的 API 使用困難。

**核心貢獻：**
- 首次在真實 API 複雜性下評估函數呼叫能力
- 識別理想化基準與真實使用之間的差距
- 設計涵蓋多種 API 困難的評估場景
- 為改善真實世界工具使用能力提供方向

**方法論：**
- 收集真實世界 API 的常見複雜性：文件不完整、參數歧義、版本不一致
- 設計對應的工具呼叫任務，模擬真實開發者遇到的情境
- 評估模型在不完美資訊下的推理與呼叫決策
- 對比理想條件與真實條件下的表現差距

**主要發現：**
- 模型在真實 API 複雜性下表現顯著劣於理想化基準
- 文件不完整是最大的效能殺手，模型無法有效推斷缺失資訊
- 參數格式歧義導致大量靜默錯誤
- 現有基準嚴重高估了 LLM 的實際工具使用能力

**研究限制：**
- API 複雜性的分類基於作者經驗，可能未窮舉所有真實世界困難類型
- 基準中的 API 為特定時間點的快照，API 的快速演進可能使基準過時
- 未探討模型透過多輪互動或文件查閱逐步解決 API 歧義的能力

**其他資訊：**
- 該基準揭示了現有評估方法的系統性樂觀偏差，對社群具有警示作用
- 研究結果暗示未來工具使用訓練應納入更多不完美資訊條件下的範例

## 綜合觀察

### 主要研究趨勢

1. **真實世界複雜性**：多篇論文（ToolGym、WildAGTEval、ReliabilityBench）強調從理想化設定轉向真實世界條件下的評估。

2. **長期互動挑戰**：RealMem、Agent Drift 揭示了代理在長期互動中的記憶與行為退化問題。

3. **多代理系統成熟化**：M3MAD-Bench、MAESTRO、Agentic Testing 顯示多代理系統評估正走向標準化。

4. **遊戲化評估**：TowerMind、MineNPC-Task 利用遊戲環境提供可控且豐富的評估場景。

5. **可信度與忠實性**：Agent Drift、Project Ariadne、Internal Representations 關注代理行為的可靠性與可解釋性。

### 關鍵缺口

- 跨語言代理能力（Arabic Prompts 僅為開端）
- 長期部署的穩定性保證
- 代理生成內容的可信度驗證（PR Inconsistency）
- 科學推理的根本性突破
