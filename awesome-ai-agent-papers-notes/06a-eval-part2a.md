## 41. MiRAGE: Multiagent Framework for Generating Multimodal Multihop QA Dataset

**論文標題：** MiRAGE: Multiagent Framework for Generating Multimodal Multihop QA Dataset (arXiv: 2601.15487)

**中文摘要：**
MiRAGE 提出一個多智能體框架，用於自動生成經過驗證的多模態多跳問答資料集。該框架透過多個專門化的 agent 協作，從不同來源收集資訊並生成需要跨模態推理的複雜問題，同時確保答案的正確性與可驗證性。

**核心貢獻：**
- 提出多智能體協作架構，自動化生成高品質多模態多跳 QA 資料集
- 整合文本與視覺資訊，生成需要跨模態推理的問題
- 內建驗證機制確保生成資料的品質與正確性
- 減少人工標註成本，同時維持資料集品質
- 支援可擴展的資料集生成流程

**方法論：**
框架採用多個專門化 agent 的協作模式：(1) 資訊收集 agent 從多個來源擷取文本與圖像資訊；(2) 問題生成 agent 設計需要多跳推理的問題；(3) 驗證 agent 確認答案的正確性與推理鏈的完整性。各 agent 之間透過結構化通訊協議進行協調，確保生成的問題確實需要整合多個資訊來源才能回答。

**主要發現：**
- 多智能體架構能有效生成高品質的多跳問答對
- 自動驗證機制能篩除不正確或品質不佳的樣本
- 生成的資料集在難度和多樣性方面優於單一模型生成的資料
- 多模態整合增加了問題的複雜度與實用性

**研究限制：**
- 多智能體框架的運算成本高昂，大規模生成資料集時 API 呼叫費用可能成為瓶頸
- 自動驗證機制的可靠性取決於驗證 agent 本身的能力，可能存在系統性盲點
- 生成的問題品質受限於來源資料的覆蓋範圍，特定領域或低資源語言的適用性未經驗證

**其他資訊：**
- 與 HotpotQA、MultiModalQA 等多跳問答資料集形成互補，但自動化程度更高
- 該框架的設計理念可推廣至其他需要多來源整合的資料集生成任務，如事實查核資料集或跨文件摘要資料集

## 42. When Agents Fail: Comprehensive Study of Bugs in LLM Agents

**論文標題：** When Agents Fail: A Comprehensive Study of Bugs in LLM Agents (arXiv: 2601.15232)

**中文摘要：**
本研究對七個主流 LLM Agent 框架中的 1,187 個 bug 報告進行系統性分析，建立了 LLM Agent 錯誤的全面分類體系。研究揭示了 agent 系統中最常見的失敗模式，並為開發者提供改進指導方針。

**核心貢獻：**
- 首次大規模系統性分析 LLM Agent 框架中的 bug 類型
- 建立包含 1,187 個 bug 報告的標註資料集
- 涵蓋七個主流 agent 框架（如 LangChain、AutoGPT 等）
- 提出 LLM Agent bug 的分類法（taxonomy）
- 識別常見失敗模式並提出防範建議

**方法論：**
研究者從七個開源 LLM Agent 框架的 GitHub issue tracker 中收集 bug 報告，進行系統性編碼與分類。分析維度包括：bug 的根本原因（root cause）、觸發條件、影響範圍、修復難度等。採用開放式編碼方法建立分類體系，並由多位研究者進行交叉驗證以確保一致性。

**主要發現：**
- Agent 特有的 bug 類型包括：提示詞相關錯誤、工具調用失敗、規劃邏輯錯誤、記憶管理問題
- 許多 bug 源於 LLM 輸出的不確定性與非結構化特性
- 工具整合是最易出錯的環節之一
- 錯誤傳播效應顯著——早期步驟的小錯誤會在後續步驟中放大
- 現有測試方法不足以覆蓋 agent 的非確定性行為

**研究限制：**
- 分析僅基於公開的 GitHub issue，可能遺漏私有專案或未被報告的 bug 類型
- 七個框架的選擇可能無法代表所有 agent 架構模式，特別是新興或專有系統
- Bug 分類依賴研究者主觀判斷，跨框架的一致性標準難以完全保證

**其他資訊：**
- 研究結果對 agent 框架開發者具有直接指導意義，可用於優先改進最脆弱的模組（如工具整合層）
- 未來可結合自動化 bug 偵測工具，建立 agent 系統的持續品質監控機制

## 43. The Why Behind the Action: Agentic Attribution

**論文標題：** The Why Behind the Action: Agentic Attribution (arXiv: 2601.15075)

**中文摘要：**
本文提出一個層次化框架，用於識別驅動 LLM Agent 行為的內部因素。該框架將 agent 的行為歸因分解為多個層次，從底層的模型偏好到高層的目標規劃，系統性地解釋「為什麼 agent 會採取某個特定動作」。

**核心貢獻：**
- 定義「Agentic Attribution」概念——解釋 agent 行動背後的原因
- 提出層次化歸因框架，涵蓋多個抽象層次
- 區分內部因素（模型知識、偏好）與外部因素（環境、指令）
- 為 agent 行為的可解釋性提供理論基礎
- 支援 agent 行為的除錯與改進

**方法論：**
框架將歸因分為多個層次：(1) 指令層——系統提示詞和使用者指令的影響；(2) 知識層——模型預訓練知識的作用；(3) 推理層——思維鏈和規劃過程的貢獻；(4) 偏好層——模型固有偏好和傾向。透過消融實驗和反事實分析，量化各層次對最終行動決策的影響程度。

**主要發現：**
- Agent 行為受多層次因素交互影響，單一歸因往往不充分
- 模型的隱含偏好對行為決策有顯著但常被忽略的影響
- 層次化歸因有助於識別 agent 行為中的系統性偏差
- 該框架可用於改進提示詞設計和 agent 架構

**研究限制：**
- 消融實驗和反事實分析的計算成本高，難以在大規模 agent 系統中即時應用
- 各歸因層次之間的交互效應難以完全解耦，存在歸因重疊的問題
- 框架的驗證主要依賴特定任務場景，跨領域和跨模型的泛化性需進一步驗證

**其他資訊：**
- 與可解釋 AI（XAI）領域的特徵歸因方法（如 SHAP、LIME）有概念上的連結，但針對 agent 的序列決策特性進行了擴展
- 對 agent 安全性研究具有重要意義——理解行為歸因有助於識別和預防非預期的有害行為

## 44. Tokenomics: Quantifying Where Tokens Are Used in Agentic SE

**論文標題：** Tokenomics: Quantifying Where Tokens Are Used in Agentic Software Engineering (arXiv: 2601.14470)

**中文摘要：**
本研究量化分析了 LLM Agent 在軟體開發生命週期各階段的 token 消耗模式。透過追蹤 agent 在不同任務類型（程式碼生成、除錯、重構等）中的 token 使用分佈，揭示了成本優化的關鍵機會點。

**核心貢獻：**
- 首次系統性量化 agentic 軟體工程中的 token 消耗分佈
- 識別各開發階段（規劃、編碼、測試、除錯）的 token 使用模式
- 區分有效 token 使用與浪費性 token 消耗
- 為 agent 系統的成本優化提供數據驅動的指導
- 建立 token 經濟學（Tokenomics）的分析框架

**方法論：**
研究設計了追蹤系統，記錄 LLM Agent 在執行軟體工程任務時每次 API 呼叫的 token 消耗。分析維度包括：輸入/輸出 token 比例、不同任務階段的消耗差異、重試和錯誤修正的額外成本、上下文視窗使用效率等。涵蓋多種軟體開發活動以確保全面性。

**主要發現：**
- 除錯和錯誤修正階段消耗的 token 比例遠超預期
- 大量 token 用於重複性的上下文提供（context stuffing）
- 規劃階段的 token 投入與任務成功率正相關
- 不同類型任務的 token 效率差異巨大
- 存在顯著的 token 浪費，主要來自不必要的重試和冗餘上下文

**研究限制：**
- Token 消耗模式可能因所使用的具體模型和 agent 框架而異，研究結論的普適性有限
- 未充分考慮不同定價模型（如快取折扣、批量定價）對實際成本的影響
- 追蹤系統本身可能引入額外開銷，且難以區分「必要的探索性 token 使用」與「真正的浪費」

**其他資訊：**
- 研究結果對企業部署 coding agent 的預算規劃具有直接參考價值，特別是在估算不同任務類型的成本比例時
- 未來可結合 token 壓縮技術（如摘要快取、漸進式上下文管理）來驗證節省效果

## 45. APEX-Agents: 480 Long-Horizon Productivity Tasks

**論文標題：** APEX-Agents (arXiv: 2601.14242)

**中文摘要：**
APEX-Agents 提出一個包含 480 個長期視野生產力任務的基準測試，涵蓋銀行、諮詢和法律三個專業領域。這些任務模擬真實工作場景中的複雜多步驟操作，用於評估 LLM Agent 在專業知識工作中的實際能力。

**核心貢獻：**
- 建立 480 個來自真實專業領域的長期視野任務基準
- 涵蓋三個高價值知識工作領域：銀行、諮詢、法律
- 任務設計反映實際工作流程的複雜性和多步驟特性
- 提供標準化的評估框架和指標
- 揭示當前 LLM Agent 在專業任務中的能力差距

**方法論：**
與領域專家合作設計任務，確保任務反映真實工作場景。每個任務包含：明確的目標描述、所需工具和資源、多個中間步驟、可驗證的最終輸出。評估指標包括任務完成率、步驟正確性、效率（步驟數和時間）、以及輸出品質的專家評分。

**主要發現：**
- 當前最先進的 LLM Agent 在長期視野專業任務上表現仍有限
- 銀行領域的數據分析任務相對容易，法律領域的推理任務最具挑戰性
- Agent 在需要維持長期上下文和多步規劃的任務中表現下降明顯
- 工具使用能力是完成專業任務的關鍵瓶頸之一
- 領域知識的缺乏是另一個重要限制因素

**研究限制：**
- 僅涵蓋三個專業領域，醫療、工程、教育等其他知識工作領域的適用性未知
- 任務的複雜度和評估標準可能受領域專家主觀判斷影響，跨評估者一致性未充分報告
- 480 個任務的規模在統計顯著性方面可能不足以支撐細粒度的能力分析

**其他資訊：**
- 與 SWE-bench 等軟體工程基準形成互補，將評估範圍擴展到非技術領域的知識工作
- 對企業 AI 部署決策具有參考價值——揭示了哪些專業任務目前適合（或不適合）委託給 AI agent

## 46. CooperBench: Why Coding Agents Cannot be Teammates Yet

**論文標題：** CooperBench: Why Coding Agents Cannot be Teammates Yet (arXiv: 2601.13295)

**中文摘要：**
CooperBench 提出一個包含 600 多個協作編程任務的基準測試，專門評估 LLM Coding Agent 的團隊協作能力。研究發現當前的 coding agent 在需要協調、溝通和分工的場景中表現遠遜於獨立工作場景。

**核心貢獻：**
- 首個專注於 coding agent 協作能力的大規模基準測試
- 包含 600+ 個需要多 agent 協作的編程任務
- 評估維度涵蓋：協調、溝通、分工、衝突解決
- 識別 coding agent 作為團隊成員的主要能力缺口
- 為未來協作式 AI 編程系統的設計提供方向

**方法論：**
設計多種協作場景：(1) 並行開發——多個 agent 同時開發不同模組；(2) 序列接力——一個 agent 的輸出作為另一個的輸入；(3) 程式碼審查——agent 間互相審查程式碼；(4) 衝突解決——處理合併衝突和設計分歧。使用自動化指標和人工評估結合的方式衡量協作品質。

**主要發現：**
- Coding agent 在獨立任務中的表現遠優於協作任務
- 主要瓶頸包括：缺乏有效的溝通協議、無法理解隊友的意圖、不善於分解和分配子任務
- Agent 傾向於獨立重做而非在他人工作基礎上繼續
- 程式碼風格和架構決策的不一致性是協作中的常見問題
- 衝突解決能力極為薄弱

**研究限制：**
- 協作場景的設計可能過度簡化真實團隊開發中的溝通模式（如非同步溝通、程式碼審查流程）
- 評估協作品質的指標尚缺乏統一標準，不同維度的權重設定可能影響結論
- 研究未考慮人機混合協作的場景，而這在實際中可能比純 agent 協作更為常見

**其他資訊：**
- 與多智能體系統（MAS）和軟體工程中的分散式開發研究有密切關聯
- 對未來 AI 輔助團隊開發工具的設計有啟示——需要在 agent 層面內建溝通協議和程式碼一致性約束

## 47. Insider Knowledge: How Much Can RAG Systems Gain from Evaluation Secrets?

**論文標題：** Insider Knowledge: How Much Can RAG Systems Gain from Evaluation Secrets? (arXiv: 2601.13227)

**中文摘要：**
本研究探討 RAG 系統如何利用（gaming）基於 nugget 的評估方法中的資訊來提升表現。研究揭示了當 RAG 系統獲取評估標準的內部資訊時，其表現會大幅提升，這引發了對現有評估方法公平性的質疑。

**核心貢獻：**
- 揭示 nugget-based 評估方法的可被利用性（gamability）
- 量化 RAG 系統在獲取評估內幕資訊後的表現提升幅度
- 提出對現有 RAG 評估範式的根本性質疑
- 分析不同程度的「內幕知識」對評估結果的影響
- 為設計更健壯的評估方法提供啟示

**方法論：**
系統性地向 RAG 系統提供不同程度的評估相關資訊：(1) 完整的 nugget 列表；(2) 部分 nugget 資訊；(3) nugget 的重要性權重；(4) 評估標準的描述。比較有無內幕資訊情況下的系統表現差異，並分析不同洩漏程度的影響。

**主要發現：**
- 即使少量的評估內幕資訊也能顯著提升 RAG 系統的評估分數
- 完整 nugget 資訊的洩漏可導致表現的大幅虛假提升
- 這種 gaming 不一定對應真實使用者體驗的改善
- 現有基於 nugget 的評估方法需要更強的防洩漏機制
- 建議採用多維度、難以被單一策略 gaming 的評估方法

**研究限制：**
- 研究的攻擊場景假設系統有直接存取評估標準的管道，在實際評測中洩漏方式可能更間接和隱蔽
- 僅針對 nugget-based 評估方法，其他評估範式（如人工評估、使用者滿意度）的可操弄性未被探討
- 提出的防範建議較為概念性，缺乏具體的防洩漏機制實作和驗證

**其他資訊：**
- 與資訊檢索領域的評估方法論研究（如 TREC 評估範式的公平性討論）密切相關
- 對 RAG 系統的競賽排名和基準測試設計者具有警示意義——需要重新思考評估資訊的隔離機制

## 48. Replayable Financial Agents: Determinism-Faithfulness Assurance

**論文標題：** Replayable Financial Agents: Determinism-Faithfulness Assurance (arXiv: 2601.15322)

**中文摘要：**
本研究探討金融領域 LLM Agent 的軌跡確定性問題，透過 74 種不同配置的實驗，測量 agent 行為在重複執行時的一致性。研究提出確定性-忠實度的度量框架，對金融 agent 的可靠性評估具有重要意義。

**核心貢獻：**
- 定義金融 agent 的「可重放性」（replayability）概念
- 提出確定性-忠實度（determinism-faithfulness）的度量框架
- 在 74 種配置下系統性測量軌跡一致性
- 識別影響 agent 確定性的關鍵因素
- 為金融領域 agent 的合規性和審計提供方法論

**方法論：**
設計金融決策任務場景，在 74 種不同的配置（包括溫度參數、提示詞變體、模型版本、隨機種子等）下重複執行 agent。測量維度包括：動作序列的一致性、決策結果的穩定性、推理過程的可復現性。使用統計方法量化軌跡間的差異程度。

**主要發現：**
- 金融 agent 的行為確定性在不同配置下差異顯著
- 溫度參數是影響確定性的最重要因素
- 即使在 temperature=0 下，仍存在非確定性行為
- 複雜的多步決策比簡單任務更難復現
- 確定性與忠實度之間存在權衡關係
- 提出改善可重放性的具體建議

**研究限制：**
- 實驗環境為模擬金融場景，真實市場的動態性和即時性可能引入額外的非確定性來源
- 74 種配置雖多但仍無法窮盡所有影響因素（如 API 版本更新、伺服器端負載變化等）
- 確定性度量框架的適用範圍侷限於可重複執行的離線分析場景，即時交易系統的適用性有限

**其他資訊：**
- 對金融監管合規具有直接意義——可重放性是審計和追責的基礎要求
- 與軟體測試中的「flaky test」問題有概念上的相似性，可借鑒相關解決方案

## 49. AEMA: Verifiable Evaluation Framework

**論文標題：** The Hierarchy of Agentic Capabilities (arXiv: 2601.09032)

**中文摘要：**

**核心貢獻：**
- 提出 agent 能力的層次結構模型
- 基於 150 個真實工作場所任務的實證分析
- 識別能力間的依賴關係（低層能力是高層的前提）
- 為 agent 系統的漸進式開發提供指導
- 建立能力評估的結構化框架

**方法論：**
從真實工作環境收集 150 個任務，分析每個任務所需的能力組合。透過因子分析和聚類方法識別能力維度，並透過依賴關係分析建立層次結構。驗證方法包括：專家評審、跨任務一致性檢驗、以及在多個 LLM Agent 上的實證測試。

**主要發現：**
- Agent 能力呈清晰的層次結構：基礎理解 → 單步執行 → 多步規劃 → 適應調整 → 自主創新
- 高層能力的展現以低層能力的穩定為前提
- 當前 LLM Agent 大多停留在中間層次
- 不同模型在層次結構中的斷層位置不同
- 該框架可用於預測 agent 在新任務上的表現

**研究限制：**
- 150 個工作場所任務的選取可能存在偏向性，未必涵蓋所有重要的 agent 能力面向
- 層次結構的嚴格線性假設可能過度簡化，某些高層能力可能不完全依賴特定低層能力
- 因子分析和聚類方法的結果可能因參數選擇而異，層次結構的穩健性需要更多交叉驗證

**其他資訊：**
- 概念設計受馬斯洛需求層次理論啟發，為 agent 能力發展提供了直覺性的理解框架
- 對 agent 系統的產品開發路線圖具有指導意義——應先確保低層能力穩定再追求高層能力

## 57. Active Evaluation of General Agents

**論文標題：** Active Evaluation of General Agents (arXiv: 2601.07651)

**中文摘要：**
本文提出一個用於通用 agent 主動評估的形式化框架。不同於被動觀察 agent 行為，主動評估透過動態調整測試條件和問題來更高效地判斷 agent 能力，減少評估所需的樣本數量。

**核心貢獻：**
- 提出主動評估（active evaluation）的形式化數學框架
- 動態調整評估策略以提高效率
- 減少全面評估 agent 所需的測試案例數量
- 適用於跨任務類型的通用 agent 評估
- 建立評估效率與準確性之間的理論界限

**方法論：**
借鑒主動學習（active learning）的思想，設計評估策略選擇演算法：根據 agent 在已完成測試上的表現，動態選擇最具資訊量的下一個測試案例。使用貝葉斯推斷更新對 agent 能力的估計，並以資訊增益最大化為目標選擇測試。

- 主動評估比隨機評估少用 30-50% 的測試案例即可達到相同的能力估計精度
- 對於能力分佈不均的 agent 效果最為顯著
- 早期測試結果對後續評估策略的引導至關重要
- 框架可以自然擴展到多維能力評估
- 在計算預算有限的情況下特別有價值

**研究限制：**
- 貝葉斯推斷的效果依賴先驗假設的合理性，錯誤的先驗可能導致早期階段的誤導性評估
- 框架假設 agent 能力在評估過程中保持穩定，但實際中 agent 可能因上下文累積而表現變化
- 資訊增益最大化的計算本身可能帶來額外開銷，在某些場景下可能抵消效率提升

**其他資訊：**
- 與電腦適性測驗（CAT）和主動學習的技術路線高度相似，是成熟方法在新領域的創新應用
- 對 agent 的持續評估和線上監控系統設計具有啟示——可在部署後高效追蹤能力變化

## 58. VirtualEnv: Embodied AI Research Platform

**論文標題：** VirtualEnv: Embodied AI Research Platform (arXiv: 2601.07553)

**中文摘要：**
VirtualEnv 是基於 Unreal Engine 5 建立的具身 AI 研究平台，專為 LLM 驅動的具身 agent 設計。該平台提供高保真度的 3D 模擬環境，支援語言指令的理解和物理互動任務的執行與評估。

**核心貢獻：**
- 基於 UE5 的高保真具身 AI 模擬平台
- 專門支援 LLM 驅動的具身 agent 研究
- 提供多樣化的室內外場景和物理互動任務
- 支援多模態感知（視覺、語言、物理）
- 標準化的任務定義和評估介面

**方法論：**
平台架構包含：(1) UE5 渲染引擎提供逼真的視覺環境；(2) 物理模擬引擎支援物體互動；(3) LLM 整合介面允許語言模型控制 agent 行為；(4) 任務管理系統定義和追蹤任務進度；(5) 評估模組自動計算任務完成度。

**主要發現：**
- UE5 的高保真度有助於縮小模擬到真實的差距（sim-to-real gap）
- LLM 驅動的 agent 在簡單導航任務上表現良好
- 需要物理推理的操作任務仍極具挑戰性
- 多模態感知整合是具身 agent 的關鍵能力
- 平台支援快速迭代和大規模並行評估

**研究限制：**
- UE5 的高計算需求限制了平台的可及性，缺乏高端 GPU 的研究團隊可能難以使用
- 模擬環境與真實世界之間仍存在 sim-to-real gap，特別是在物體材質、光照變化和物理互動的細節上
- 平台的任務設計可能偏向視覺導航類，對細緻操控和觸覺相關任務的支援不足

**其他資訊：**
- 與 AI2-THOR、Habitat 等具身 AI 平台形成競爭和互補關係，UE5 的優勢在於視覺保真度
- 對機器人學的 sim-to-real 遷移研究有潛在價值，高保真模擬可減少真實環境訓練的需求

## 59. FROAV: RAG Observation and Agent Verification

**論文標題：** FROAV: RAG Observation and Agent Verification (arXiv: 2601.07504)

**中文摘要：**
FROAV 提出一個視覺化工作流編排框架，結合 LLM-as-a-Judge 機制對 RAG agent 進行觀察和驗證。該框架能夠追蹤 RAG agent 的完整執行流程，並使用 LLM 評判器自動識別執行中的問題和錯誤。

**核心貢獻：**
- 視覺化的 RAG agent 工作流編排和觀察框架
- 整合 LLM-as-a-Judge 進行自動驗證
- 支援執行軌跡的即時監控和事後分析
- 識別 RAG 管線中的常見失敗模式
- 提供可操作的改進建議

**方法論：**
框架由三部分組成：(1) 工作流編排器——定義和管理 RAG agent 的執行步驟；(2) 觀察器——記錄每個步驟的輸入、輸出和中間狀態；(3) 驗證器——使用 LLM-as-a-Judge 評估每個步驟和最終輸出的品質。視覺化介面允許研究者直觀地檢查和分析 agent 行為。

**主要發現：**
- 視覺化工作流大幅提升了 RAG 系統除錯效率
- LLM-as-a-Judge 能有效識別檢索品質問題和生成錯誤
- 最常見的失敗模式包括：相關文件未被檢索、上下文過載、答案幻覺
- 自動驗證與人工判斷的一致性達到可用水準
- 框架可整合到現有 RAG 開發流程中

**研究限制：**
- LLM-as-a-Judge 本身可能存在偏見和錯誤，自動驗證的可靠性上限受限於評判模型的能力
- 視覺化工具的有效性評估可能受使用者經驗水平影響，新手和專家的體驗差異未被充分探討
- 框架主要針對離線分析，對生產環境中即時監控的延遲和資源消耗未做深入評估

**其他資訊：**
- 與 LangSmith、Weights & Biases 等商業可觀測性工具有功能重疊，但更聚焦於學術研究的可重現性
- 對 RAG 系統的工程實踐具有直接價值——提供了系統性識別和修復管線問題的方法論

## 60. Lost in the Noise: Reasoning Models Fail with Distractors

**論文標題：** Lost in the Noise: Reasoning Models Fail with Distractors (arXiv: 2601.07226)

**中文摘要：**
本研究系統性地評估了推理模型（reasoning models）在面對上下文中干擾資訊時的魯棒性。研究發現，即使是最先進的推理模型，在輸入中包含不相關或誤導性資訊時，表現也會顯著下降，揭示了當前模型在實際應用中的脆弱性。

**核心貢獻：**
- 系統性評估推理模型對干擾資訊的魯棒性
- 設計多種類型的干擾因素（noise/distractors）
- 量化干擾對推理品質的影響程度
- 比較不同模型的抗干擾能力差異
- 為提升模型魯棒性提供方向

**方法論：**
在標準推理任務（數學推理、邏輯推理、閱讀理解等）的輸入中系統性地注入不同類型的干擾資訊：(1) 無關資訊——與任務不相關的文本段落；(2) 誤導性資訊——看似相關但實際錯誤的內容；(3) 矛盾資訊——與正確線索衝突的訊號。控制干擾的數量、位置、相似度等變數，測量表現下降程度。

**主要發現：**
- 所有測試的推理模型在面對干擾時表現下降
- 誤導性資訊比無關資訊造成更大的表現降低
- 干擾資訊與正確資訊越相似，影響越嚴重
- 推理鏈越長的任務受干擾的影響越大
- 模型規模的增大並不能保證魯棒性的提升
- 這對 RAG 系統和長上下文應用具有重要警示意義

**研究限制：**
- 干擾資訊的注入方式較為人為，真實場景中的「雜訊」分佈和形態可能更加多樣和自然
- 研究主要聚焦於文本推理任務，對多模態推理場景中干擾的影響探討不足
- 未深入分析模型的注意力機制如何被干擾資訊影響，缺乏機制性解釋

**其他資訊：**
- 對 RAG 系統的檢索品質要求提出了更高標準——檢索到的無關文件不僅是浪費，更可能主動損害推理品質
- 與對抗性攻擊研究有關聯，但聚焦於自然情境下的「非故意干擾」，對實際部署的穩健性設計更具指導意義
