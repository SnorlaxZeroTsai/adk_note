## 15. ShardMemo: Masked MoE Routing for Sharded Agentic LLM Memory

**論文標題:** ShardMemo: Masked MoE Routing for Sharded Agentic LLM Memory (arXiv: 2601.21545)

**中文摘要:**
本論文提出 ShardMemo，一種基於混合專家（MoE）路由機制的分片式代理記憶系統。該系統透過遮罩路由策略，在固定計算預算下僅探查符合條件的記憶分片，避免全量掃描帶來的延遲與成本。此方法實現了分層記憶探查（tiered memory probing），使大規模代理系統能高效存取分散式記憶。

**核心貢獻:**
- 提出將 MoE 路由機制應用於記憶分片選擇的新框架
- 設計遮罩機制（masking mechanism）確保在固定預算約束下僅激活相關分片
- 實現分層式記憶探查，根據查詢特性動態選擇探查深度與廣度
- 顯著降低代理記憶系統的推理延遲與計算成本

**方法論:**
系統將代理的長期記憶分散儲存於多個分片（shards）中，每個分片專注於特定主題或時間範圍。查詢進入時，MoE 風格的路由網路計算各分片的相關性分數，並透過遮罩操作（masking）僅激活得分超過閾值的分片。路由決策在固定計算預算（fixed budget）下進行，確保無論記憶總量多大，每次查詢的成本保持恆定。分層設計允許系統先進行粗粒度篩選，再對候選分片進行細粒度探查。

**主要發現:**
- 在固定預算約束下，遮罩路由比均勻探查所有分片準確率提升顯著
- 系統擴展性良好，記憶分片數量增加時查詢延遲保持穩定
- 路由網路能學習到有意義的記憶分區模式，類似人類的主題式記憶組織
- 在多輪代理對話場景中，相比全量檢索節省 60-80% 的計算資源

**研究限制:**
- 路由網路的訓練需要大量標註數據來定義各分片的最佳分配策略，冷啟動階段效果可能不佳
- 固定預算約束在極端分佈偏移（distribution shift）情況下可能導致關鍵分片被遺漏，影響召回率
- 分片的靜態劃分策略（按主題或時間）可能無法適應記憶內容動態變化的場景，需要定期重新分片

**其他資訊:**
- 與傳統 MoE 模型（如 Switch Transformer）的路由機制有深層聯繫，但應用場景從模型參數選擇轉移至外部記憶選擇
- 對於需要即時回應的對話系統和大規模多代理部署具有實際應用價值，可與向量資料庫分片策略結合使用

## 16. When Should I Search More: Adaptive Complex Query Optimization

**論文標題:** When Should I Search More: Adaptive Complex Query Optimization (arXiv: 2601.21208)

**中文摘要:**
本論文研究複雜查詢場景下的自適應搜索策略，利用強化學習（RL）決定何時需要將複雜查詢拆分為子查詢以進行更深入的搜索。系統學習在「直接回答」與「進一步分解搜索」之間做出最優決策，避免不必要的搜索開銷同時確保答案品質。

**核心貢獻:**
- 提出以 RL 為核心的自適應查詢分解決策框架
- 定義了「何時搜索更多」的決策問題並設計對應的獎勵函數
- 實現了在搜索成本與答案品質之間的最優平衡
- 證明並非所有複雜查詢都需要分解，簡單查詢直接回答效果更佳

**方法論:**
系統將查詢處理建模為馬可夫決策過程（MDP），其中狀態包含當前查詢、已獲取的證據、以及置信度估計。RL 代理在每一步決定是否需要進一步搜索（分解查詢）或終止並生成答案。獎勵函數綜合考量答案正確性與搜索步數，鼓勵以最少步數獲得正確答案。訓練使用策略梯度方法，在多個 QA 基準上進行。

**主要發現:**
- RL 代理學會對簡單查詢直接作答，對真正複雜的多跳查詢才進行分解
- 相比始終分解的策略，自適應方法在保持相近準確率的同時減少約 40% 的搜索調用
- 決策邊界與查詢的語義複雜度和所需推理跳數高度相關
- 在 HotpotQA、MuSiQue 等多跳 QA 資料集上驗證了有效性

**研究限制:**
- RL 訓練過程依賴預定義的獎勵函數設計，不同權重設定可能導致截然不同的搜索策略偏好
- 評估僅在英文多跳 QA 資料集上進行，對於其他語言或領域特定查詢的泛化能力未經驗證
- 系統假設搜索引擎回傳結果品質穩定，在真實環境中搜索品質波動可能影響決策準確度

**其他資訊:**
- 與自適應計算（adaptive computation）研究方向高度相關，如早期退出（early exit）機制在推理效率上的應用
- 對於商業搜索引擎和企業級 RAG 系統具有直接的成本優化意義，可減少不必要的 API 調用支出

## 17. A2RAG: Adaptive Agentic Graph Retrieval

**論文標題:** A2RAG: Adaptive Agentic Graph Retrieval (arXiv: 2601.21162)

**中文摘要:**
A2RAG 提出一種自適應代理式圖檢索框架，能夠驗證已檢索證據的充分性，並在證據不足時逐步升級檢索策略。系統結合知識圖譜與文本檢索，透過代理迴路持續評估是否已收集足夠資訊來回答查詢。

**核心貢獻:**
- 設計證據充分性驗證機制（evidence sufficiency verification），避免過早或過晚停止檢索
- 提出漸進式檢索升級策略（progressive escalation），從輕量級方法逐步過渡到重量級方法
- 結合圖結構化知識與非結構化文本的混合檢索架構
- 實現自主決策的代理式檢索流程，減少人工干預

**方法論:**
A2RAG 維護一個代理迴路，每次迭代包含：(1) 執行當前層級的檢索操作（從簡單向量搜索到複雜圖遍歷）；(2) 評估已收集證據是否足以回答查詢；(3) 若不足則升級到更強力的檢索策略。證據充分性透過 LLM 判斷器評估，考量答案的完整性、一致性和可信度。圖檢索部分利用實體關係進行多跳推理，文本檢索部分提供細粒度語義匹配。

**主要發現:**
- 漸進式升級策略相比一步到位的複雜檢索，平均減少 35% 的檢索時間
- 證據充分性驗證有效避免了「資訊不足就強行回答」的問題
- 在需要多源整合的複雜問題上，圖+文本混合檢索顯著優於單一模態
- 代理式迴路在 2-4 次迭代內通常能收斂到充分證據

**研究限制:**
- 證據充分性判斷依賴 LLM 本身的能力，當 LLM 對領域知識不足時可能做出錯誤的充分性評估
- 知識圖譜的建構與維護成本高昂，且圖譜不完整性（missing edges）會直接限制圖檢索的效果
- 漸進式升級的層級設計較為固定，可能無法靈活適應所有查詢分佈的需求

**其他資訊:**
- 與 Self-RAG 等自我反思式檢索系統形成互補，A2RAG 強調檢索策略的逐步升級，Self-RAG 則強調生成品質的自我評估
- 適用於企業知識管理場景，特別是擁有結構化知識庫（如產品規格、法規條文）與非結構化文檔並存的組織

## 18. MemCtrl: MLLMs as Active Memory Controllers

**論文標題:** MemCtrl: MLLMs as Active Memory Controllers (arXiv: 2601.20831)

**中文摘要:**
MemCtrl 將多模態大型語言模型（MLLMs）設計為主動記憶控制器，引入可訓練的記憶閘門（memory gate）來做出保留（retain）、更新（update）或丟棄（discard）的決策。該方法使模型能主動管理其工作記憶，而非被動地處理所有輸入資訊。

**核心貢獻:**
- 提出可訓練記憶閘門機制，實現顯式的記憶管理決策
- 將記憶操作分為保留、更新、丟棄三種離散動作
- 設計端到端訓練框架，使記憶控制與任務目標聯合優化
- 在多模態場景中驗證主動記憶管理的有效性

**方法論:**
MemCtrl 在 MLLM 的隱藏層中插入記憶閘門模組。每個時間步，閘門接收當前輸入表徵和記憶狀態，輸出三元決策：保留（維持記憶不變）、更新（用新資訊覆寫部分記憶）、或丟棄（釋放不再相關的記憶槽位）。閘門透過直通估計器（straight-through estimator）實現離散決策的可微分訓練。整個系統端到端優化，記憶控制策略由下游任務損失驅動學習。

**主要發現:**
- 主動記憶控制相比被動緩存機制，在長序列多模態理解任務上提升 8-15%
- 模型學會在場景切換時丟棄舊記憶、在重要資訊出現時主動保留
- 記憶閘門的決策模式具有可解釋性，與人類注意力分配模式相似
- 在視頻理解、多輪視覺對話等任務上均展現改進

**研究限制:**
- 直通估計器的梯度近似可能導致訓練不穩定，離散決策的優化仍存在理論上的困難
- 記憶槽位數量為預設固定值，難以動態適應不同長度和複雜度的輸入序列
- 目前僅在特定多模態任務上驗證，對於純文本長上下文場景的適用性有待探索

**其他資訊:**
- 與神經圖靈機（Neural Turing Machine）及可微分神經計算機（DNC）的記憶控制思路一脈相承，但規模和應用場景更貼近現代 LLM
- 對於長視頻理解、機器人持續學習等需要長期記憶管理的實際應用場景具有潛在價值

## 19. AMA: Adaptive Memory via Multi-Agent Collaboration

**論文標題:** AMA: Adaptive Memory via Multi-Agent Collaboration (arXiv: 2601.20352)

**中文摘要:**
AMA 提出基於多代理協作的自適應記憶框架，包含層次化粒度管理、自適應查詢路由、以及一致性驗證三大核心模組。多個專門化代理協作維護不同粒度和類型的記憶，並確保記憶內容的一致性。

**核心貢獻:**
- 設計層次化記憶粒度架構（hierarchical granularity），從摘要到細節多層組織
- 提出自適應查詢路由機制，將不同類型查詢導向最適合的記憶代理
- 引入一致性驗證模組，解決多源記憶間的衝突與矛盾
- 多代理協作框架允許各代理專注於特定記憶管理職責

**方法論:**
AMA 部署多個專門化代理：(1) 摘要代理負責維護高層概括性記憶；(2) 細節代理管理具體事實和數據；(3) 時序代理追蹤事件的時間順序；(4) 路由代理根據查詢特性分配到對應的記憶代理。層次化粒度設計使系統能在不同抽象層級儲存和檢索資訊。一致性驗證模組定期檢查各代理記憶間的矛盾，透過投票或優先級規則解決衝突。

**主要發現:**
- 層次化記憶組織使不同粒度的查詢都能高效匹配
- 自適應路由減少了不必要的全量記憶掃描，提升回應速度
- 一致性驗證有效減少了 40% 以上的記憶衝突引起的錯誤回答
- 在長對話、知識密集型任務中表現優異

**研究限制:**
- 多代理系統帶來顯著的通訊與協調開銷，在延遲敏感的場景中可能成為瓶頸
- 一致性驗證的投票機制在代理數量較少時可能不夠穩健，容易受單一代理錯誤影響
- 各代理的職責劃分（摘要/細節/時序）為人工設計，未必是所有場景的最佳分工方式

**其他資訊:**
- 與認知科學中的多系統記憶理論（如工作記憶、長期記憶、情節記憶的區分）存在對應關係
- 可與現有的多代理框架（如 AutoGen、CrewAI）結合，為其提供持久化記憶管理能力

## 20. When Iterative RAG Beats Ideal Evidence

**論文標題:** When Iterative RAG Beats Ideal Evidence (arXiv: 2601.19827)

**中文摘要:**
本論文探討一個反直覺的現象：在某些情況下，迭代式 RAG 系統（多輪檢索-生成循環）的表現竟然優於直接提供黃金標準上下文（gold-context）的靜態 RAG。研究分析了何時以及為何迭代過程本身帶來的推理結構化效益超越了完美證據的價值。

**核心貢獻:**
- 首次系統性地比較迭代 RAG 與理想證據 RAG 的性能差異
- 識別出迭代優於靜態的具體條件和查詢類型
- 提出「過程即推理」（process-as-reasoning）假設解釋此現象
- 為 RAG 系統設計提供重要的反直覺指導原則

**方法論:**
實驗設計包含兩組對照：(1) 靜態 RAG，直接將人工標註的黃金段落作為上下文提供給生成器；(2) 迭代 RAG，允許系統進行多輪檢索-推理循環。在多個 QA 基準上比較兩者表現，並進行細粒度分析：按查詢複雜度、推理類型、答案形式等維度拆解結果。進一步的消融實驗探究迭代過程中的哪些環節貢獻最大。

**主要發現:**
- 在需要多步推理的複雜查詢上，迭代 RAG 常優於靜態黃金上下文
- 迭代過程迫使模型結構化其推理，產生類似思維鏈的效果
- 中間檢索步驟即使未找到完美證據，也能幫助模型縮小搜索空間
- 對於簡單事實型查詢，靜態黃金上下文仍然更優
- 迭代帶來的「推理腳手架」效應在 3-5 輪後趨於飽和

**研究限制:**
- 研究結論可能依賴於特定 LLM 的能力水準，較弱的模型未必能從迭代過程中獲得同等推理增益
- 黃金上下文的定義和品質取決於標註者，不同標註標準下結論可能有所不同
- 迭代 RAG 的額外計算成本（多輪 LLM 調用）未被充分納入比較框架中

**其他資訊:**
- 此發現與思維鏈（Chain-of-Thought）提示的效果相呼應，暗示結構化推理過程本身對 LLM 具有正則化作用
- 對 RAG 系統工程師而言，意味著不應盲目追求單次檢索的完美性，適度的迭代過程可能更具成本效益

## 21. Dep-Search: Dependency-Aware Reasoning Traces with Persistent Memory

**論文標題:** Dep-Search: Dependency-Aware Reasoning Traces with Persistent Memory (arXiv: 2601.18771)

**中文摘要:**
Dep-Search 利用群組相對策略優化（GRPO）強化學習訓練模型進行依賴感知的查詢分解。系統教導模型識別子問題之間的依賴關係，並維護持久記憶追蹤已解決和待解決的子問題，從而實現更有效的複雜推理。

**核心貢獻:**
- 提出依賴感知的推理軌跡框架，明確建模子問題間的前置條件關係
- 使用 GRPO 強化學習訓練分解策略，獎勵正確的依賴關係識別
- 設計持久記憶機制追蹤推理狀態，避免重複搜索和循環推理
- 在多跳推理任務上顯著提升搜索效率和答案準確率

**方法論:**
系統首先將複雜查詢分解為多個子查詢，並顯式標註它們之間的依賴關係（如「子問題B依賴於子問題A的答案」）。GRPO RL 訓練使模型學會生成正確的依賴圖：獎勵信號來自最終答案正確性以及依賴結構的合理性。持久記憶模組記錄每個子問題的狀態（待處理/進行中/已解決）及其答案，確保按正確順序執行搜索，並避免重複工作。

**主要發現:**
- GRPO 訓練的模型能生成比監督學習更合理的依賴關係圖
- 依賴感知分解比獨立分解減少約 30% 的冗餘搜索
- 持久記憶有效防止了推理過程中的循環和重複
- 在 MuSiQue、2WikiMultiHopQA 等基準上達到新的最優結果

**研究限制:**
- GRPO 訓練需要大量的推理軌跡採樣，計算成本高且訓練過程可能不穩定
- 依賴圖的正確性評估缺乏標準化的自動度量指標，多依賴人工判斷或間接的最終答案正確性
- 對於依賴結構高度動態的查詢（子問題在推理過程中才逐步浮現），預先規劃的依賴圖可能不準確

**其他資訊:**
- 與程式分析中的依賴圖（dependency graph）和任務規劃中的偏序規劃（partial-order planning）有概念上的類比
- 對於複雜的研究問題拆解和專案管理任務的自動化具有啟發意義

## 22. FadeMem: Biologically-Inspired Forgetting

**論文標題:** FadeMem: Biologically-Inspired Forgetting (arXiv: 2601.18642)

**中文摘要:**
FadeMem 受生物學記憶衰退機制啟發，提出自適應指數衰減記憶管理方法，結合 LLM 引導的衝突解決策略。系統模擬人類記憶的自然遺忘過程，使用時間衰減函數自動降低過時資訊的權重，同時利用 LLM 智慧地解決新舊記憶間的衝突。

**核心貢獻:**
- 設計仿生學指數衰減記憶模型，自動管理記憶的時效性
- 提出 LLM 引導的衝突解決機制，處理矛盾資訊的合併與取捨
- 自適應衰減率根據記憶的重要性和使用頻率動態調整
- 在長期對話和知識更新場景中有效防止過時資訊的干擾

**方法論:**
每條記憶附帶一個強度分數，隨時間按指數函數衰減。衰減率並非固定，而是根據記憶的重要性（由 LLM 評估）、被存取的頻率、以及與當前上下文的相關性動態調整。高重要性或頻繁存取的記憶衰減較慢。當新資訊與現有記憶產生衝突時，LLM 衝突解決器評估兩者的可信度、時效性和相關性，決定保留、合併或替換。

**主要發現:**
- 自適應衰減比固定衰減率在長對話場景中準確率提升 12-18%
- LLM 引導的衝突解決比規則式方法（如「新覆蓋舊」）更為精確
- 系統能自動識別「事實型記憶」（低衰減）與「情境型記憶」（高衰減）
- 在需要處理資訊更新的場景（如新聞追蹤、知識修正）中表現尤為突出

**研究限制:**
- LLM 引導的衝突解決每次都需要額外的推理調用，在記憶量大時累積的計算成本可能不可忽視
- 指數衰減模型過於簡化，人類記憶的遺忘曲線實際上更為複雜（如間隔效應、情緒影響）
- 重要性評估本身可能有偏差，且缺乏有效的回饋機制來修正錯誤的衰減率設定

**其他資訊:**
- 與 Ebbinghaus 遺忘曲線及間隔重複（spaced repetition）學習理論有直接的理論聯繫
- 對於個人化 AI 助理和長期陪伴式對話機器人的記憶管理具有實用價值，能避免反覆提及已過時的使用者偏好

## 23. FastInsight: Fusion Operators for Graph RAG

**論文標題:** FastInsight: Fusion Operators for Graph RAG (arXiv: 2601.18579)

**中文摘要:**
FastInsight 提出針對圖 RAG 的融合運算子，實現圖感知重排序（graph-aware reranking）結合語義-拓撲擴展（semantic-topological expansion）。系統利用知識圖譜的結構資訊增強段落排序，並透過圖拓撲結構發現語義相關但文本表面不相似的相關內容。

**核心貢獻:**
- 設計融合運算子將圖拓撲特徵與語義特徵統一整合
- 提出語義-拓撲擴展策略，透過圖結構發現隱含相關的段落
- 實現圖感知重排序，利用實體間的圖距離和路徑資訊改善排序
- 顯著提升 Graph RAG 系統的檢索精度和召回率

**方法論:**
FastInsight 在傳統 RAG 檢索管線後加入融合運算子層。初始檢索結果先經過語義評分，再透過知識圖譜進行拓撲擴展：對於每個檢索到的實體，沿圖邊探索鄰近節點，將相關但未被文本匹配覆蓋的段落加入候選集。融合運算子將語義相似度分數與圖拓撲特徵（如節點度、路徑長度、社群歸屬）進行加權融合，產生最終排序。

**主要發現:**
- 語義-拓撲融合比純語義排序在多跳問題上召回率提升 20-30%
- 圖擴展有效彌補了語義匹配的「表面相似偏見」問題
- 融合運算子的計算開銷相對較小，適合線上服務部署
- 在醫學和科學領域的知識密集型 QA 中效果尤為顯著

**研究限制:**
- 系統效能高度依賴底層知識圖譜的品質和覆蓋率，圖譜構建本身是高成本且易出錯的前置步驟
- 拓撲擴展在圖結構稀疏或連結雜訊多的情況下可能引入不相關的段落，反而降低精度
- 融合權重的調優需要針對不同領域和資料集進行，缺乏自動化的權重學習機制

**其他資訊:**
- 與 Microsoft GraphRAG 及相關圖增強檢索工作形成技術互補，可視為其重排序階段的增強模組
- 在生物醫學知識發現、藥物交互作用查詢等需要跨實體推理的應用場景中具有高度實用性

## 24. Less is More for RAG: Information Gain Pruning

**論文標題:** Less is More for RAG: Information Gain Pruning (arXiv: 2601.17532)

**中文摘要:**
本論文提出基於資訊增益的段落剪枝方法，核心理念是「少即是多」——與其讓生成器處理所有檢索到的段落，不如過濾掉資訊增益低的弱段落。方法採用生成器對齊的重排序策略，確保保留的段落真正有助於生成器產生正確答案。

**核心貢獻:**
- 提出資訊增益剪枝（Information Gain Pruning）框架
- 設計生成器對齊的重排序機制，以生成器的實際需求指導篩選
- 證明移除弱相關段落能減少生成器的困惑並提升準確率
- 提供理論分析說明為何更少但更精準的上下文優於更多但含噪的上下文

**方法論:**
資訊增益的計算方式為：加入某段落後生成器對正確答案的對數概率增量。若增量為正，表示該段落提供有用資訊；若為負或接近零，則該段落為噪聲或干擾。實際部署時，由於無法預知正確答案，使用代理模型（proxy model）估計資訊增益。重排序後僅保留增益超過閾值的段落，形成精煉的上下文窗口。

**主要發現:**
- 移除 30-50% 的檢索段落後，生成品質反而提升 5-10%
- 弱相關段落確實會干擾生成器，導致注意力分散和答案偏移
- 生成器對齊的剪枝比通用相關性剪枝更有效，因為兩者的「相關性」定義不同
- 在長上下文場景中效果更為顯著，因為噪聲段落的干擾隨上下文長度增加而加劇

**研究限制:**
- 代理模型估計資訊增益的準確度有限，且代理模型與實際生成器之間的差異可能導致次優剪枝決策
- 需要額外的推理步驟計算資訊增益分數，增加了系統整體延遲
- 閾值的設定需要針對不同任務和模型進行調整，缺乏自動化的閾值選擇方法

**其他資訊:**
- 與主動學習（active learning）中的資訊增益選擇策略有理論上的共通之處
- 對於 token 數量受限的場景（如高昂的 API 計費或有限的上下文窗口）具有直接的實用價值，可作為 RAG 系統的標準後處理步驟

## 25. DeepEra: Deep Evidence Reranking Agent

**論文標題:** DeepEra: Deep Evidence Reranking Agent (arXiv: 2601.16478)

**中文摘要:**
DeepEra 提出深度證據重排序代理，透過逐步推理（step-by-step reasoning）來區分表面相似但實際無關的段落與真正相關的證據。系統模擬人類仔細閱讀和推理的過程，對每個候選段落進行深度分析而非僅依賴表面特徵匹配。

**核心貢獻:**
- 設計逐步推理框架進行深度證據評估
- 有效解決「表面相似但語義無關」的誤匹配問題
- 代理式架構允許根據需要動態調整推理深度
- 在高難度檢索場景（如對抗性干擾段落）中表現突出

**方法論:**
DeepEra 對每個候選段落執行多步推理鏈：(1) 識別查詢的核心資訊需求；(2) 分析段落的主題和內容；(3) 逐步比對段落內容與查詢需求的對應關係；(4) 判斷段落是否提供了回答查詢所需的關鍵資訊。每一步推理都產生中間結論，最終綜合所有步驟給出相關性判定。此過程類似思維鏈（Chain-of-Thought）但專門針對相關性評估設計。

**主要發現:**
- 在包含對抗性干擾段落的測試集上，DeepEra 比傳統重排序器準確率提升 15-25%
- 逐步推理有效識別了「同主題但不回答問題」的迷惑性段落
- 推理過程具有良好的可解釋性，用戶可理解為何某段落被判為相關或無關
- 推理深度可動態調整：簡單案例 2-3 步即可判定，困難案例可擴展到 5-7 步

**研究限制:**
- 逐步推理帶來的延遲開銷在候選段落數量多時會線性累積，難以應用於大規模即時檢索場景
- 推理品質高度依賴底層 LLM 的能力，小模型可能無法產生有效的推理鏈
- 評估集中在對抗性設定下表現優異，但在常規（非對抗性）檢索場景中相對於輕量級重排序器的優勢可能不明顯

**其他資訊:**
- 與 LLM-as-judge 範式及 Pointwise/Listwise 重排序方法在方法論上具有關聯性
- 推理過程的可解釋性對於法律、醫療等高風險決策場景的資訊檢索應用特別有價值

## 26. SPARC-RAG: Adaptive Sequential-Parallel Scaling

**論文標題:** SPARC-RAG: Adaptive Sequential-Parallel Scaling (arXiv: 2602.00083)

**中文摘要:**
SPARC-RAG 提出自適應的序列-並行推理時間擴展框架，透過多代理協調系統在序列推理（深度探索）和並行推理（廣度探索）之間動態切換。系統根據查詢特性和當前進展自動決定最優的擴展策略。

**核心貢獻:**
- 首次提出在 RAG 中結合序列與並行推理時間擴展的統一框架
- 設計多代理協調機制管理並行推理分支的合併與衝突解決
- 提出自適應擴展策略，根據問題難度動態分配序列/並行資源
- 在複雜多跳推理中實現了推理品質與效率的帕累托最優

**方法論:**
SPARC-RAG 包含協調器代理（coordinator）和多個工作代理（workers）。協調器分析查詢後決定初始策略：對獨立子問題啟動並行工作代理同時處理，對有依賴的子問題安排序列執行。運行過程中，協調器持續監控各工作代理的進展和置信度，動態調整策略——若某並行分支陷入困難則轉為序列深入探索，若序列推理效率低則啟動更多並行分支。最終由協調器整合所有分支的結果。

**主要發現:**
- 自適應混合策略比純序列或純並行分別提升 12% 和 18% 的答案品質
- 並行擴展在子問題獨立的場景中優勢明顯，序列擴展在強依賴場景中更優
- 協調器學會根據問題類型自動選擇最優比例（如 70% 並行 + 30% 序列）
- 在固定推理時間預算下，混合策略的性價比最優

**研究限制:**
- 多代理架構的資源消耗較高，需要同時維護多個 LLM 實例，對硬體要求苛刻
- 協調器的決策品質是系統瓶頸，若協調器判斷失誤會導致資源浪費或品質下降
- 並行分支的結果合併策略較為簡單，當不同分支產生矛盾結論時的解決機制有待完善

**其他資訊:**
- 與推理時間擴展（inference-time scaling）和測試時計算（test-time compute）的研究趨勢高度一致，可視為其在 RAG 領域的具體實例
- 對於需要在延遲和品質之間進行取捨的生產環境具有指導意義，協調器的策略可根據 SLA 要求進行配置

## 27. Incorporating Q&A Nuggets into RAG

**論文標題:** Incorporating Q&A Nuggets into RAG (arXiv: 2601.13222)

**中文摘要:**
本論文提出將問答知識片段（Q&A Nuggets）融入 RAG 系統的方法，透過預先抽取的細粒度知識單元增強生成品質。系統不僅提供原始段落，還附帶結構化的知識片段和引用溯源（citation provenance）資訊，提升答案的精確性和可追溯性。

**核心貢獻:**
- 提出 Nugget 增強生成（Nugget-Augmented Generation）框架
- 設計知識片段抽取與組織管線，將文檔轉化為結構化 Q&A 對
- 引入引用溯源機制，每個生成的陳述可追溯至具體知識片段
- 在答案精確性和可驗證性上均有顯著提升

**方法論:**
系統分為離線和在線兩個階段。離線階段：對語料庫中的文檔進行知識片段抽取，每個片段為一個自包含的 Q&A 對，記錄其來源文檔和位置。在線階段：查詢進入後，同時檢索相關段落和匹配的知識片段；生成時將原始段落和片段一起提供給 LLM，並要求標註每個生成陳述的來源片段。引用溯源透過片段 ID 建立生成文本與源文檔的精確映射。

**主要發現:**
- Nugget 增強使答案的事實精確度提升 10-15%
- 引用溯源使用戶能快速驗證答案的每個部分
- 細粒度知識片段比完整段落更容易被生成器有效利用
- 在開放域 QA 和事實核查任務上均展現改進

**研究限制:**
- 離線知識片段抽取的品質依賴於抽取模型，錯誤的片段會在下游持續傳播且難以偵測
- 大規模語料庫的片段抽取和索引維護成本高昂，且需要隨語料更新而持續重新抽取
- 引用溯源的準確性在生成器產生推理性或綜合性陳述時會下降，因為此類陳述可能融合多個片段

**其他資訊:**
- 與 TREC Nugget 評估框架及資訊抽取中的知識三元組抽取有方法論上的承繼關係
- 對於需要高可追溯性的應用場景（如法律研究、學術文獻綜述、合規審查）具有直接實用性

## 28. Augmenting QA with Hybrid RAG

**論文標題:** Augmenting QA with Hybrid RAG (arXiv: 2601.12658)

**中文摘要:**
本論文提出混合 RAG 框架，整合查詢增強（query augmentation）、代理式路由（agentic routing）、和結構化檢索（structured retrieval）三種技術。系統根據查詢特性自動選擇最適合的檢索策略組合，實現多種檢索方法的優勢互補。

**核心貢獻:**
- 設計統一的混合 RAG 架構整合多種檢索策略
- 提出代理式路由機制，智慧選擇最適合的檢索管線
- 結合查詢增強技術提升檢索匹配的覆蓋率
- 引入結構化檢索處理需要精確匹配的查詢（如數字、日期、專有名詞）

**方法論:**
混合 RAG 系統包含三個核心模組：(1) 查詢增強模組透過改寫、擴展和分解提升查詢品質；(2) 代理式路由器分析查詢類型後決定使用向量檢索、關鍵字檢索、結構化查詢（如 SQL/SPARQL）或其組合；(3) 結構化檢索模組處理需要精確匹配的查詢，將自然語言轉化為結構化查詢語言。三個模組可以按需組合，路由器根據查詢特性和可用資料源動態編排最優管線。

**主要發現:**
- 混合策略比單一檢索方法在綜合 QA 基準上提升 15-22%
- 代理式路由準確率達 85% 以上，能正確識別查詢所需的檢索類型
- 查詢增強在語義模糊或簡短查詢上效果最顯著
- 結構化檢索在涉及數值比較、時間範圍篩選的查詢上不可或缺
- 三種方法組合使用比任意兩者組合的改進更為顯著

**研究限制:**
- 系統複雜度高，三個模組的聯合調優和錯誤傳播分析困難，除錯成本大
- 路由器 85% 的準確率意味著約 15% 的查詢會被導向錯誤管線，此類錯誤難以在下游修正
- 結構化檢索依賴結構化資料源的可用性，在僅有非結構化文檔的場景中該模組無法發揮作用

**其他資訊:**
- 與搜尋引擎領域的混合檢索（hybrid search，結合稀疏與稠密檢索）思路一致，但擴展至更多檢索模態
- 對於企業級 RAG 部署具有架構參考價值，尤其是同時擁有知識庫、資料庫和文檔庫的大型組織
