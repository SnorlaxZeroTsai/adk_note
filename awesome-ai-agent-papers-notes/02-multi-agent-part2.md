## 28. Mixture-of-Models: Unifying Heterogeneous Agents via N-Way Self-Evaluating Deliberation (arXiv: 2601.16863)

### 中文摘要
本論文提出 Mixture-of-Models (MoM) 框架，在推論時動態整合異質 LLM 智能體的能力。透過動態專長仲介機制（dynamic expertise broker）選擇最適合的模型組合，並以二次投票（quadratic voting）共識機制融合多模型輸出，實現比單一模型更優越的效能。

### 核心貢獻
- 提出運行時（runtime）MoM 架構，無需額外訓練即可組合異質模型
- 設計動態專長仲介（Dynamic Expertise Broker），根據查詢特徵自動路由至最適模型子集
- 引入 N-Way 自我評估審議（N-Way Self-Evaluating Deliberation），讓模型互相評分
- 採用二次投票（Quadratic Voting）共識機制，平衡少數模型的強信號與多數模型的弱信號
- 無需訓練或微調，完全在推論階段運作

### 方法論
- **動態專長仲介**：根據輸入查詢的特徵向量，動態評估各模型在該類型任務上的預期表現，選擇最相關的模型子集參與推論
- **N-Way 自我評估**：每個參與的模型生成回答後，所有模型（包括自己）對每個回答進行品質評分
- **二次投票共識**：採用二次投票機制（投票成本與票數的平方成正比），防止單一模型壟斷決策，同時允許對高信心判斷投入更多影響力
- **異質模型統一介面**：設計統一的通訊協議，使不同架構、不同能力的模型能在同一框架下協作

### 主要發現
- MoM 在多個基準測試上超越最佳單一模型，證明異質整合的價值
- 二次投票機制顯著優於簡單多數決或加權平均
- 動態路由有效降低計算成本，避免不必要的模型調用
- 自我評估審議能識別並抑制模型幻覺，提升答案可靠性

### 研究限制
- 多模型推論的計算成本仍然偏高，即使有動態路由機制，同時調用多個大型模型的 API 費用和延遲在實際生產環境中可能難以承受
- 自我評估的可靠性取決於模型本身的判斷能力，當所有參與模型在特定領域均表現不佳時，互評機制可能無法有效識別錯誤
- 評估主要集中於標準基準測試，對於開放式生成任務或特定領域專業問題的泛化能力尚未充分驗證

### 其他資訊
- 本研究與 LLM 路由（LLM routing）和模型集成（model ensemble）文獻密切相關，可視為將傳統集成學習思想擴展至異質大型語言模型的嘗試
- 對於需要高可靠性輸出的應用場景（如醫療、法律），MoM 的多重驗證機制具有實際部署潛力，但需進一步研究如何在成本與品質間取得平衡

## 29. Multi-Agent Constraint Factorization Reveals Latent Invariant Solution Structure (arXiv: 2601.15077)

### 中文摘要
本論文運用運算子理論（operator theory）正式化解釋為何多智能體系統能找到單一智能體無法觸及的不變解（invariant solutions）。透過約束因式分解（constraint factorization），揭示多智能體協作中隱含的不變解結構，提供了多智能體系統優越性的理論基礎。

### 核心貢獻
- 提供嚴格的數學框架，證明多智能體系統能存取單一智能體結構性無法達到的解空間
- 引入約束因式分解方法，將複雜約束分解為可由不同智能體獨立處理的子約束
- 揭示潛在不變解結構（latent invariant solution structure）的存在
- 建立多智能體協作的理論必要性條件
- 連結運算子代數與多智能體系統設計

### 方法論
- **運算子理論框架**：將多智能體系統的求解過程建模為希爾伯特空間（Hilbert space）上的運算子作用
- **約束因式分解**：將全局約束分解為局部約束的張量積（tensor product），每個因子對應一個智能體的行動空間
- **不變子空間分析**：證明因式分解後的聯合運算子具有單一運算子所不具備的不變子空間
- **可達性證明**：形式化證明存在一類問題，其解只存在於多智能體聯合空間的不變子空間中

### 主要發現
- 多智能體系統的優勢不僅是實務上的效率提升，更是結構性的能力差異
- 約束因式分解揭示了問題的隱含對稱性，這些對稱性只有在多智能體架構下才可見
- 單一智能體在某些問題類別上存在原理性的限制，無法透過增加計算資源克服
- 此理論框架可指導多智能體系統的架構設計，確定何時需要多智能體協作

### 研究限制
- 理論框架高度抽象，從運算子理論到實際多智能體系統設計之間的橋接尚不明確，實務指導性有限
- 約束因式分解假設問題結構已知且可分解，但現實中許多問題的約束結構難以先驗確定
- 證明主要適用於特定數學結構的問題類別，對於非結構化或模糊約束的實際應用場景，理論保證可能不成立

### 其他資訊
- 本研究為多智能體系統提供了類似於分散式計算中 CAP 定理的基礎性理論結果，有助於理解多智能體協作的本質優勢
- 未來可探索如何自動識別問題的約束結構，進而決定最適合的智能體分工方式，將理論成果轉化為系統設計工具

## 30. MAS-Orchestra: Multi-Agent Reasoning Through Holistic Orchestration (arXiv: 2601.14652)

### 中文摘要
MAS-Orchestra 提出一個訓練時（training-time）框架，將多智能體系統的編排問題形式化為函數呼叫強化學習（function-calling RL）。透過將編排決策視為一系列函數呼叫動作，使用 RL 優化編排策略，實現多智能體推理的整體性協調。

### 核心貢獻
- 將多智能體編排（orchestration）形式化為函數呼叫 RL 問題
- 提出整體性編排框架，統一處理智能體選擇、任務分配與結果整合
- 設計訓練時優化機制，學習最佳編排策略
- 支援動態智能體團隊組成與任務依賴圖

### 方法論
- **函數呼叫 RL 形式化**：將編排器（orchestrator）的每個決策建模為一次函數呼叫，函數參數對應智能體選擇和任務規格
- **狀態空間設計**：包含當前任務進度、各智能體能力描述、歷史互動記錄
- **動作空間定義**：函數呼叫的組合空間，包括調用哪個智能體、傳遞什麼上下文、如何整合結果
- **獎勵信號設計**：結合任務完成品質、計算效率、智能體利用率的多目標獎勵
- **策略優化**：使用 PPO 或類似演算法訓練編排策略網路

### 主要發現
- 學習式編排顯著優於基於規則的靜態編排策略
- 函數呼叫抽象使得編排決策自然對應 LLM 的工具使用能力
- 訓練後的編排器展現出任務分解與並行化的湧現能力
- 在複雜多步推理任務上，整體性編排相比貪心式委派獲得更好表現

### 研究限制
- 訓練編排策略需要大量的多智能體互動數據和獎勵信號，資料收集成本高且難以涵蓋所有任務類型
- 函數呼叫 RL 的動作空間隨智能體數量和任務規格的組合呈指數增長，擴展至大規模智能體團隊時面臨可擴展性挑戰
- PPO 等 RL 演算法的訓練穩定性問題可能影響編排策略的可靠性，且訓練後的策略對未見過的智能體組合泛化能力有待驗證

### 其他資訊
- 本方法將多智能體編排與 LLM 的工具使用（tool use）能力統一化，為現有支援函數呼叫的 LLM 提供了直接可用的編排升級路徑
- 可與自動化提示工程（automated prompt engineering）結合，探索編排策略與智能體提示詞的聯合優化

## 31. MASCOT: Multi-Agent Socio-Collaborative Companion Systems (arXiv: 2601.14230)

### 中文摘要
MASCOT 提出多智能體社交協作夥伴系統，透過雙層優化（bi-level optimization）對齊智能體人格（personas），並利用 RLAIF（Reinforcement Learning from AI Feedback）確保智能體行為符合設定的社交角色。系統設計用於需要多個具有不同人格特徵的智能體協同服務使用者的場景。

### 核心貢獻
- 提出多智能體夥伴系統架構，每個智能體具有獨立且一致的人格
- 設計雙層優化框架：外層優化人格一致性，內層優化任務完成品質
- 整合 RLAIF 進行人格對齊，無需大量人工標註
- 實現多智能體間的社交互動協議，支援角色間的動態合作與衝突

### 方法論
- **雙層優化（Bi-level Optimization）**：
  - 外層：優化人格參數，確保智能體行為與設定人格一致
  - 內層：在固定人格約束下，優化任務導向的回應品質
- **RLAIF 對齊**：使用 AI 評審員評估智能體回應是否符合其人格設定，生成偏好數據進行強化學習
- **社交互動協議**：定義智能體間的訊息傳遞格式、輪流機制與衝突解決規則
- **人格表示學習**：將人格特徵編碼為可學習的向量，支援精細的人格控制

### 主要發現
- 雙層優化有效解耦人格一致性與任務品質，避免兩者互相干擾
- RLAIF 能以低成本實現高品質的人格對齊
- 具有互補人格的多智能體團隊比同質團隊表現更好
- 系統在長期互動中維持人格一致性的能力顯著優於基線方法

### 研究限制
- 人格一致性的評估高度主觀，RLAIF 中 AI 評審員的偏好可能無法完全代表真實使用者的感知
- 雙層優化的計算開銷較大，外層與內層的交替優化收斂速度慢，限制了快速迭代實驗的可能性
- 社交互動協議的設計偏向英語文化語境，跨文化場景下的人格表現和適當性尚未深入探討

### 其他資訊
- 本研究對虛擬助手、教育陪伴、心理健康支持等需要持續且一致人格互動的應用場景具有直接參考價值
- 人格對齊技術可與角色扮演遊戲中的 NPC 設計結合，探索更自然的多角色互動體驗

## 32. If You Want Coherence, Orchestrate a Team of Rivals (arXiv: 2601.14351)

### 中文摘要
本論文提出「對手團隊」（Team-of-Rivals）架構，透過讓多個持不同觀點的智能體進行結構化辯論，配合遠端程式碼執行器（remote code executor）驗證事實宣稱，以提升生成內容的連貫性和準確性。此架構受歷史上林肯內閣「對手團隊」治理理念啟發。

### 核心貢獻
- 提出 Team-of-Rivals 多智能體架構，刻意引入觀點多樣性
- 整合遠端程式碼執行器作為事實查核機制
- 設計結構化辯論協議，確保分歧能有效收斂至高品質共識
- 證明對抗性協作比同質化團隊產生更連貫的輸出

### 方法論
- **對手團隊設計**：刻意為不同智能體設定對立或互補的立場、風格和優先序
- **結構化辯論**：多輪辯論結構，包含立場陳述、質疑挑戰、反駁回應、共識形成四個階段
- **遠端程式碼執行器**：辯論中涉及的事實宣稱（如數值計算、邏輯推理）透過程式碼執行進行驗證
- **連貫性仲裁**：最終由仲裁者智能體整合辯論結果，產出連貫且經過充分論證的最終回應
- **分歧管理**：設計明確的分歧解決機制，避免無限辯論

### 主要發現
- 刻意引入對立觀點顯著提升推理深度和輸出品質
- 程式碼執行器有效消除事實性錯誤，降低幻覺率
- Team-of-Rivals 在需要多角度分析的任務上表現特別突出
- 結構化辯論比自由形式討論更有效率且品質更高

### 研究限制
- 多輪辯論顯著增加推論的 token 消耗和延遲，對於需要即時回應的應用場景實用性有限
- 程式碼執行器僅能驗證可形式化的事實宣稱（如數值計算），對於主觀判斷、道德推理等無法程式化驗證的領域效果有限
- 辯論收斂機制的設計需要精細調整，過早收斂可能錯失重要觀點，過晚收斂則浪費計算資源

### 其他資訊
- 此架構與「AI 辯論」（AI debate）安全性研究方向高度相關，可作為 AI 對齊（alignment）的輔助機制，透過對抗性檢驗降低有害輸出風險
- 適合應用於報告撰寫、政策分析、投資決策等需要多角度論證的高風險決策場景

## 33. The Orchestration of Multi-Agent Systems: Architectures, Protocols, Enterprise Adoption (arXiv: 2601.13671)

### 中文摘要
本論文提出統一框架，整合 MCP（Model Context Protocol）和 A2A（Agent-to-Agent）通訊協議，全面探討多智能體系統的架構模式、通訊協議與企業採用策略。文章橋接學術研究與產業實踐，為企業級多智能體系統部署提供系統性指導。

### 核心貢獻
- 提出整合 MCP 和 A2A 協議的統一多智能體編排框架
- 系統性分類多智能體架構模式（集中式、分散式、混合式）
- 深入分析企業採用多智能體系統的挑戰與解決方案
- 提供從原型到生產部署的實踐指南

### 方法論
- **架構分類學**：建立多智能體系統架構的完整分類體系，涵蓋拓撲結構、控制流、通訊模式
- **協議整合**：
  - MCP：標準化模型與外部工具/資源的互動介面
  - A2A：定義智能體間的直接通訊協議
  - 統一層：設計橋接兩者的中間層
- **企業就緒性分析**：評估安全性、可擴展性、可觀測性、合規性等企業需求
- **案例研究**：分析多個產業場景的多智能體系統部署實例

### 主要發現
- MCP 和 A2A 是互補而非競爭的協議，需要統一整合
- 企業採用的主要障礙是可觀測性和可控性，而非技術能力
- 混合架構（結合集中式控制與分散式執行）最適合企業場景
- 標準化協議對於多智能體系統的互操作性和規模化至關重要

### 研究限制
- 作為綜述性論文，提出的統一框架主要停留在概念層次，缺乏大規模生產環境的實證驗證
- MCP 和 A2A 協議仍在快速演進中，論文中的分析可能隨協議更新而過時
- 企業案例研究的代表性有限，不同產業（金融、醫療、製造等）的特殊需求未能逐一深入分析

### 其他資訊
- 本論文對於正在評估多智能體系統企業部署的技術決策者具有高度實用價值，可作為技術選型的參考框架
- 隨著 MCP 成為業界事實標準，本文提出的 MCP-A2A 統一層概念可能影響未來開源框架的設計方向

## 34. MARO: Learning Stronger Reasoning from Social Interaction (arXiv: 2601.12323)

### 中文摘要
MARO（Multi-Agent Reward Optimization）提出從社交互動中學習更強推理能力的方法，將社交場景中的整體結果分解為針對個別行為的獎勵信號。此方法使智能體能從多智能體互動的經驗中提取細粒度的學習信號，提升單一智能體的推理能力。

### 核心貢獻
- 提出多智能體獎勵優化方法，從社交互動中提取個體學習信號
- 設計社交結果分解機制，將集體結果歸因至個體行為
- 證明社交互動經驗能有效提升智能體的推理能力
- 提供從多智能體經驗遷移至單智能體能力的途徑

### 方法論
- **社交互動環境**：構建多智能體互動場景，智能體需要透過推理和溝通完成協作任務
- **結果分解（Outcome Decomposition）**：使用信用分配（credit assignment）技術，將團隊整體表現分解為每個智能體每個行為的貢獻
- **Per-Behavior 獎勵信號**：為每個推理步驟、溝通動作生成精細的獎勵信號
- **獎勵優化訓練**：使用分解後的獎勵信號，透過 RL 優化個別智能體的推理策略
- **社交到個體遷移**：將社交互動中學到的推理模式內化為智能體的自主能力

### 主要發現
- 從社交互動分解出的獎勵信號比稀疏的結果獎勵更有效
- 智能體在社交環境中學到的推理能力可遷移至個體推理任務
- 多樣化的社交互動伙伴能提供更豐富的學習信號
- MARO 在數學推理和邏輯推理基準上顯著超越純自我博弈方法

### 研究限制
- 信用分配（credit assignment）在複雜多輪互動中的準確性難以保證，錯誤的歸因可能導致學習到次優策略

## 36. Do We Always Need Query-Level Workflows? (arXiv: 2601.11147)

### 方法論
- **策略對比實驗**：
  - 靜態工作流：預定義的固定執行流程
  - 查詢級動態工作流：為每個查詢即時生成定製工作流
  - 混合策略：先判斷查詢複雜度，簡單查詢用靜態流程，複雜查詢用動態流程

- 動態工作流的額外延遲和成本在簡單任務上不具正當性
- 混合策略在品質和效率間取得最佳平衡
- 判斷查詢複雜度的分類器本身可以很輕量

### 研究限制
- 查詢複雜度的界定缺乏普適標準，60-70% 的比例高度依賴具體任務領域和評估資料集的選擇
- 複雜度分類器的誤判（將複雜查詢判為簡單）可能導致品質嚴重下降，而此類錯誤的成本不對稱
- 實驗設定可能未涵蓋足夠多樣的真實使用者查詢分布，結論的通用性需在更多場景中驗證

### 其他資訊
- 本研究對多智能體系統的工程實踐具有重要指導意義：不應盲目追求最複雜的架構，而應根據實際需求選擇適當的方案複雜度
- 混合策略的思想可推廣至其他 AI 系統設計決策，如模型大小選擇（小模型處理簡單請求、大模型處理困難請求）

## 37. Learning Latency-Aware Orchestration for Parallel Multi-Agent Systems (arXiv: 2601.10560)

### 中文摘要
本論文解決並行多智能體系統中的延遲感知編排問題，目標是優化關鍵執行路徑（critical execution path）。當多個智能體可以並行執行時，整體延遲由最長路徑決定；本方法學習如何分配任務以最小化此關鍵路徑延遲。

### 核心貢獻
- 形式化並行多智能體系統的延遲優化問題
- 提出學習式延遲感知編排策略
- 考慮智能體異質性和任務依賴關係的聯合優化
- 設計關鍵路徑識別與最小化演算法

### 方法論
- **問題形式化**：將任務建模為有向無環圖（DAG），節點為子任務，邊為依賴關係，每個節點有與智能體相關的預估執行時間
- **延遲模型**：考慮智能體的計算延遲、通訊延遲、排隊延遲的綜合效應
- **關鍵路徑優化**：識別 DAG 中的最長路徑，優先優化瓶頸節點的分配
- **學習式調度**：使用強化學習訓練調度策略，考慮歷史延遲數據和當前系統負載
- **動態重規劃**：運行時監測實際延遲，動態調整後續任務分配

### 主要發現
- 延遲感知編排相比延遲無關的品質優先編排，顯著降低端到端延遲
- 關鍵路徑優化比均勻負載平衡更有效
- 學習式方法能適應不同智能體的延遲特性和波動模式
- 在高並行度場景下，延遲優化帶來的收益更加顯著

### 研究限制
- 延遲模型假設各智能體的執行時間可預測，但實際中 LLM 推論延遲受負載、網路狀況等因素影響，波動較大
- 延遲優化可能犧牲輸出品質（將任務分配給更快但能力較弱的智能體），品質與延遲的權衡策略需要更細緻的控制
- DAG 模型要求任務依賴關係預先明確，但在動態推理過程中，依賴關係可能隨中間結果變化而改變

### 其他資訊
- 本方法對於即時性要求高的應用（如客戶服務、即時翻譯、互動式程式碼生成）具有直接的工程價值
- 可與雲端服務的自動擴展機制結合，在延遲壓力大時動態增加智能體實例，形成彈性多智能體系統

- 多智能體設定下的測試時學習效果優於單智能體
- 方法在數學推理和程式設計任務上展現一致的提升
- 計算開銷相比傳統多次採樣方法更具效率

### 研究限制
- 測試時 RL 的效果高度依賴結構化經驗的品質和相關性，當遇到與經驗庫差異極大的新問題時，引導效果可能大打折扣
- 多智能體審議的計算成本仍然顯著高於單次推論，在資源受限環境中的適用性有限
- 動態獎勵信號的設計依賴一致性和互驗等啟發式指標，可能在某些推理模式（如需要反直覺思維的問題）上產生誤導

### 其他資訊
- 本方法展示了一種不修改模型權重即可提升推理能力的途徑，對於無法進行模型微調的場景（如使用商業 API）特別有價值
- 測試時適應的思想可與檢索增強生成（RAG）結合，將經驗注入機制與動態知識檢索統一，探索更通用的測試時增強框架
