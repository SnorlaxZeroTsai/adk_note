## 名詞速查表

首次閱讀時可能會遇到不熟悉的術語，這裡快速解釋：

| 術語 | 一句話解釋 |
|------|-----------|
| Token | LLM 處理文字的最小單位，約等於一個英文單字或 1-2 個中文字 |
| Provider | 提供 LLM API 的供應商（如 OpenAI、Anthropic、Google） |
| Completion | 向 LLM 發送一段文字，取得它的回覆。這個動作叫 "completion" |
| Streaming | 不等 LLM 全部回完，邊生成邊接收（像打字一樣逐字出現） |
| Context Window | LLM 一次能處理的最大 token 數（記憶長度） |
| Proxy / Gateway | 站在你和供應商之間的中間伺服器，負責路由、認證、限流 |
| Router | 決定一個請求應該發到哪個供應商 / 哪個 API Key 的元件 |
| Fallback | 主要的供應商失敗後，自動切換到備用的機制 |
| MCP | Model Context Protocol，讓 AI 助手調用外部工具的標準協議 |

## 環境準備（可選）

如果你想邊讀邊跑程式碼：

```bash
# 1. Clone LiteLLM 原始碼
git clone https://github.com/BerriAI/litellm.git
cd litellm

# 2. 建立虛擬環境
python -m venv .venv && source .venv/bin/activate

# 3. 安裝開發依賴
pip install -e ".[dev]"

# 4. 確認安裝成功
python -c "import litellm; print(litellm.__version__)"
```
